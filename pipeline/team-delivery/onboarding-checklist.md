# 클라이언트 온보딩 체크리스트

> 계약 체결 후 → 딜리버리 시작 전 수집해야 할 정보

---

## 킥오프 미팅 전 — 클라이언트에게 요청할 것

### 필수 정보
- [ ] 비즈니스 한 줄 소개
- [ ] 타겟 고객 설명 (누구한테 팔고 있는지)
- [ ] 현재 사용 중인 SNS 채널 + 계정 링크
- [ ] 웹사이트/포트폴리오 링크 (있다면)
- [ ] 현재 고객 유입 경로 (어디서 고객이 오는지)
- [ ] 월 평균 신규 고객 수
- [ ] 원하는 목표 (월 몇 명 신규 고객)

### 선택 정보 (있으면 도움)
- [ ] 기존 고객 리스트 (이메일/연락처)
- [ ] 과거 마케팅 시도 + 결과
- [ ] 경쟁사 3곳 (본인이 생각하는)
- [ ] 브랜드 톤/스타일 가이드
- [ ] 기존에 쓰는 도구 목록 (Notion, 캘린더, 이메일 등)

### 접근 권한 (필요시)
- [ ] SNS 계정 접근 (콘텐츠 자동화 시)
- [ ] Notion 워크스페이스 초대 (CRM 구축 시)
- [ ] 이메일 도구 접근 (이메일 시퀀스 시)

---

## 킥오프 미팅 아젠다 (30분)

| 시간 | 항목 | 내용 |
|---|---|---|
| 0~5분 | 인사 + 진행 설명 | 7일 프로세스 전체 설명 |
| 5~15분 | 비즈니스 심층 이해 | 사전 정보 기반 추가 질문 |
| 15~25분 | 파이프라인 설계 합의 | 4개 모듈 중 우선순위 결정 |
| 25~30분 | 다음 스텝 + 일정 | 중간 체크인, 인수인계 일정 잡기 |

## 킥오프 후 내부 프로세스

### 1. Research 에이전트에 리서치 요청
```
"클라이언트 온보딩 리서치 실행
- 업종: {업종}
- 타겟 고객: {타겟}
- 채널: {현재 채널}
(team-research/playbook.md의 클라이언트 온보딩 리서치 프롬프트 사용)"
```

### 2. 파이프라인 설계서 작성
```
프롬프트: "아래 클라이언트 정보를 바탕으로 세일즈 파이프라인 설계서를 작성해줘.

클라이언트: {이름}
업종: {업종}
타겟 고객: {타겟}
현재 채널: {채널}
목표: {목표}
리서치 결과: {Research 에이전트 결과}

설계서 포함 항목:
1. 리드 발굴 전략 (채널 + 방법)
2. 콘텐츠 전략 (카테고리 + 빈도 + 채널)
3. 아웃리치 전략 (메시지 + 시퀀스)
4. CRM 구조 (Notion DB 설계)
5. 일일/주간 루틴 (30분 이내)"
```

### 3. Delivery 팀 구축 시작
- `pipeline/team-delivery/clients/{이름}/` 폴더 생성
- 설계서 저장
- 7일 타임라인 시작

## 완료 체크리스트 (인수인계 전)

- [ ] 리드 발굴 시스템 테스트 완료 (최소 10명 리드 수집)
- [ ] 콘텐츠 1주일치 샘플 생성 완료
- [ ] 아웃리치 템플릿 3종 + 개인화 가이드 완료
- [ ] Notion CRM 세팅 + 샘플 데이터 입력 완료
- [ ] 인수인계 가이드 문서 완성
- [ ] 일일 루틴 30분 이내 확인
- [ ] 클라이언트 테스트 실행 확인
