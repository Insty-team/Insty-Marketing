# Team Delivery 플레이북

> 목표: 클라이언트 파이프라인을 1주일 안에 구축 & 인수인계

---

## 딜리버리 프로세스 (7일)

### Day 1-2: 리서치 + 설계
1. Research 팀의 클라이언트 리서치 결과 확인
2. 클라이언트 킥오프 미팅 (30분)
   - 현재 세일즈 프로세스 파악
   - 목표 고객 프로필 확인
   - 기존에 쓰는 도구 확인
   - 원하는 결과물 합의
3. 파이프라인 설계서 작성

### Day 3-5: 구축
4. 리드 발굴 시스템 구축
5. 콘텐츠 자동화 세팅
6. 아웃리치 시퀀스 작성
7. CRM (Notion DB) 세팅

### Day 6-7: 테스트 + 인수인계
8. 전체 파이프라인 테스트 실행
9. 인수인계 미팅 (1시간)
   - 각 시스템 사용법 설명
   - 가이드 문서 전달
   - Q&A
10. 2주 후 체크인 일정 잡기

## 구축 모듈별 가이드

### 모듈 1: 리드 발굴 시스템

```
프롬프트: "{클라이언트 업종}의 잠재 고객을 자동으로 발굴하는 시스템을 설계해줘.

클라이언트 정보:
- 업종: {업종}
- 타겟 고객: {타겟}
- 현재 채널: {기존 채널}

설계해야 할 것:
1. 타겟 고객이 활동하는 채널 3-5개
2. 각 채널에서 리드를 찾는 방법 (키워드, 해시태그, 그룹)
3. 리드 자격 기준 (어떤 사람이 진짜 잠재 고객인지)
4. 주간 리드 수집 루틴 (시간 투자 최소화)
5. Notion DB에 저장할 리드 정보 구조"
```

### 모듈 2: 콘텐츠 자동화

```
프롬프트: "{클라이언트 업종}에 맞는 SNS 콘텐츠 자동화 시스템을 만들어줘.

구성:
1. 콘텐츠 카테고리 5개 (요일별 로테이션)
2. 각 카테고리별 프롬프트 템플릿
3. 톤/스타일 가이드
4. 해시태그 세트 (채널별)
5. 일주일치 샘플 콘텐츠 생성"
```

### 모듈 3: 아웃리치 시퀀스

```
프롬프트: "{클라이언트 업종}에서 {타겟 고객}에게 보낼 아웃리치 시퀀스를 만들어줘.

시퀀스 구조:
1. 1차 DM/이메일 (첫 접촉) — 3가지 변형
2. 2차 팔로업 (48시간 후, 무응답 시)
3. 3차 가치 제공 (5일 후, 무응답 시) — 무료 리소스 첨부
4. 관심 표현 시 → 미팅 예약 메시지
5. 각 메시지에 개인화 포인트 표시 {이렇게}"
```

### 모듈 4: CRM (Notion DB)

Notion DB 구조:
```
데이터베이스: {클라이언트명} 세일즈 파이프라인

| 필드 | 타입 | 용도 |
|---|---|---|
| 이름 | Title | 잠재 고객 이름 |
| 상태 | Select | 리드/접촉/대화중/미팅/제안/성사/보류 |
| 채널 | Select | 유입 채널 |
| 연락처 | Text | 이메일/SNS |
| 첫 연락일 | Date | |
| 마지막 연락일 | Date | |
| 다음 액션 | Text | 해야 할 일 |
| 예상 매출 | Number | |
| 태그 | Multi-select | 분류 |
| 메모 | Text | 대화 내용 등 |
```
