# Pipeline 작업 규칙

## Phase 1 컨텍스트
- 한국 1인 사업자 대상 AI 세일즈 파이프라인 구축 대행
- 4주 안에 첫 매출 목표
- 패키지: Starter ₩50만 / Growth ₩100만 / Premium ₩150만

## 팀별 에이전트 프롬프팅 패턴

### 역할 부여 (Role Prompting)
팀별 에이전트에게 반드시 역할을 먼저 부여:
- Research: "한국 1인 사업자 시장 전문 리서치 애널리스트"
- Sales: "B2B 소규모 고객 전문 세일즈 전략가"
- Content: "듀얼 트랙 콘텐츠 마케터 (한국 서비스 + 영문 빌드인퍼블릭)"
- Delivery: "AI 자동화 구축 전문 프로젝트 매니저"

### 작업 요청 시 필수 포함 사항
1. **목표**: 이 작업의 구체적 결과물
2. **제약**: 바꾸면 안 되는 것
3. **검증**: 완료 기준 (예: "DM 응답률 5% 이상 달성 가능한 메시지")
4. **출력 형식**: 마크다운, JSON, 표 등 형식 지정

### 단계별 사고 유도 (Chain of Thought)
복잡한 분석 작업 시 반드시 단계별로 요청:
```
1. 현재 상황 분석
2. 가능한 접근법 3가지 나열
3. 각 접근법의 장단점
4. 최선의 방법 선택 후 실행
```
