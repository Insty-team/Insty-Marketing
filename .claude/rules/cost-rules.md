# 비용 최적화 규칙

## 모델 선택
- 단순 형식 변환, JSON 생성: Haiku 사용
- 일반 코딩, 콘텐츠 작성: Sonnet 사용
- 아키텍처 결정, 복잡한 전략 분석: Opus 사용

## 컨텍스트 관리
- 작업 전환 시 `/clear` 실행
- verbose 출력 작업은 서브에이전트에 위임 (결과만 반환)
- `/compact` 시 "코드 변경사항과 진행 상황에 집중해서 요약" 지시

## 서브에이전트 비용 절감
- Explore 에이전트: Haiku 모델 사용 (읽기 전용이므로 충분)
- 병렬 분석 시 서브에이전트 활용하되 3개 이하로 제한
- 서브에이전트 프롬프트는 구체적으로 (모호하면 토큰 낭비)

## 미사용 MCP 서버
- 사용하지 않는 MCP 서버는 `/mcp`에서 비활성화
- CLI 도구 (gh, docker)가 MCP보다 컨텍스트 효율적
