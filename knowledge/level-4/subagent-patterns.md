# 서브에이전트 패턴

서브에이전트는 **메인 대화에서 분리된 독립 에이전트**에게 작업을 위임하는 시스템입니다. 이 아키텍처는 메인 컨텍스트를 보호하면서 병렬 처리를 가능하게 합니다.

## 빌트인 서브에이전트

Claude Code에는 6가지 기본 서브에이전트가 내장되어 있습니다:

| 서브에이전트 | 모델 | 도구 | 용도 |
|-----------|------|------|------|
| Explore | Haiku | 읽기 전용 | 파일 탐색, 코드 검색 |
| Plan | 상속 | 읽기 전용 | 플랜 모드에서 코드베이스 조사 |
| general-purpose | 상속 | 전체 | 복잡한 리서치, 멀티스텝 작업 |
| Bash | 상속 | - | 터미널 명령 실행 |
| statusline-setup | Sonnet | - | `/statusline` 설정 시 사용 |
| Claude Code Guide | Haiku | - | Claude Code 기능 질문 응답 |

## 핵심 특성

**컨텍스트 분리**: 결과만 메인 대화에 반환되어 탐색 결과가 컨텍스트를 차지하지 않습니다.

**중첩 제한**: "서브에이전트는 **다른 서브에이전트를 생성할 수 없습니다**."

## 포그라운드 vs 백그라운드

포그라운드는 기본 동작으로 완료까지 메인 대화를 차단합니다. 백그라운드는 `Ctrl+B`로 전환 가능하며 사전 승인된 권한만 사용합니다.

## 실전 활용

- 병렬 코드 분석
- 대규모 리팩토링
- 멀티스텝 워크플로우

지속적인 병렬 작업이 필요하면 서브에이전트 대신 Agent Teams를 고려하세요.
