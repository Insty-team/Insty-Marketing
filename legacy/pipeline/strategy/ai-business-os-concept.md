# AI Business OS 컨셉 — 점진적 자율화 모델

> 2026-03-11 아이디어 기록. 서비스 설계 시 참고.

## 핵심 아이디어

1인 사업자에게 "AI Business OS"를 세팅해주되, **처음부터 풀 자동화가 아니라 점진적으로 자율성을 높여가는 모델**.

## 비유: 테슬라 자율주행 모델

테슬라가 ADAS(Advanced Driver Assistant System)에서 시작해 FSD(Full Self-Driving)로 진화한 것과 같은 접근.

| 테슬라 | AI Business OS |
|--------|---------------|
| ADAS — 차선 유지, 경고 | Phase 1 — "이런 문제가 있습니다" 알림 |
| Autopilot — 고속도로 자동 주행 | Phase 2 — 검증된 업무 자동 실행 |
| FSD — 목적지만 입력 | Phase 3 — 비즈니스 파이프라인 자율 운영 |

핵심은 동일: **데이터(맥락)를 쌓으면서 점진적으로 자율성을 확대**. 처음부터 FSD를 파는 게 아니라, ADAS로 신뢰를 쌓고 데이터를 모으는 것.

## 진화 단계

```
Phase 1: 보조 (Assistant)
  ↓
Phase 2: 반자동 (Co-pilot)
  ↓
Phase 3: 자율 (Autopilot)
```

### Phase 1: 보조 — "이런 문제가 있을 수 있습니다"
- 고객의 비즈니스 정보를 **맥락 DB**에 차곡차곡 축적
- 각 파트별 AI Agent가 맥락을 이해하고:
  - "이런 문제점이 있을 수 있습니다"
  - "이런 조치를 취하는 것을 권장드립니다"
  - "이것을 제가 해드릴까요?"
- 사람이 승인하면 실행, 아니면 대안 제시
- **신뢰 구축 기간** — AI가 맥락을 학습하고, 고객이 AI를 신뢰하는 과정

### Phase 2: 반자동 — "이건 제가 알아서 했습니다"
- 반복적이고 검증된 작업은 자동 실행 (보고만 함)
- 새로운/중요한 판단은 여전히 사람에게 질문
- 예: "매주 월요일 SNS 포스팅은 제가 알아서 올립니다. 이번 주 콘텐츠 확인하실래요?"

### Phase 3: 자율 — "파이프라인이 돌아가고 있습니다"
- 안정된 파이프라인은 fully AI agent가 자율 운영
- 사람은 대시보드로 모니터링 + 전략적 결정만
- 이상 감지 시에만 알림

## 성공의 2가지 핵심

### 1. 정확한 맥락 저장
- **텍스트 기반, 파트별 카테고리 분류** 형태로 저장
- 사업 운영에 필요한 파트별로 knowledge를 구조화:

```
business-knowledge/
├── product/        — 제품/서비스 정보, 가격, 기능, 로드맵
├── customer/       — ICP, 페르소나, 고객 피드백, 구매 패턴
├── marketing/      — 브랜드 가이드, 채널 전략, 콘텐츠 톤앤매너
├── sales/          — 세일즈 스크립트, 오퍼, 클로징 패턴, 파이프라인 현황
├── operations/     — 프로세스, SOP, 도구 세팅, 비용 구조
├── finance/        — 매출, 비용, 목표, 인보이스 이력
└── decisions/      — 과거 의사결정 로그 (ADR 스타일, 왜 그렇게 했는지)
```

- 각 파트의 Agent가 해당 카테고리의 텍스트를 맥락으로 읽고 판단
- Agent가 "왜"를 이해할 수 있어야 함 — 단순 데이터가 아니라 의사결정 맥락
- 처음엔 간단한 마크다운/텍스트, 나중에 필요 시 벡터 DB로 확장

### 2. AI Agent에게 도구 쥐어주기
- 각 Agent에게 실행 가능한 도구(tool) 제공
  - 마케팅 Agent → SNS API, 이메일 발송, 콘텐츠 생성
  - 세일즈 Agent → CRM 접근, DM 발송, 제안서 생성
  - 운영 Agent → 스케줄 관리, 인보이스, 리포트 생성
- MCP (Model Context Protocol) 활용하면 도구 연결이 표준화됨

## 서비스 제공 방식 (아이디어)

```
Week 1-2: 비즈니스 진단 + 맥락 DB 구축
Week 3-4: Phase 1 Agent 세팅 (보조 모드)
Month 2-3: Phase 2로 점진적 전환 (검증된 작업 자동화)
Month 4+: Phase 3 안정화 (자율 파이프라인)
```

## 우리 Insty 파이프라인과의 연결
- 지금 만들고 있는 콘텐츠 실험 파이프라인이 바로 Phase 3의 예시
  - 매일 자동으로 레퍼런스 발견 → 분석 → 가이드 생성 → Notion 저장
  - 사람은 "오늘 뭐 찍을까?" → Notion 열면 가이드 3개 대기 중
- 이걸 고객에게도 세팅해주는 것이 서비스
