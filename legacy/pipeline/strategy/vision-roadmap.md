# Insty Vision Roadmap

---

## 미션

> **"누구나 자신이 하고 싶은 일로 살 수 있는 세상을 만든다"**

도구는 바뀐다. AI든 그 다음 무엇이든.
변하지 않는 건 사람들이 **자기가 원하는 일을 하며 살고 싶다**는 것.
Insty는 그 사이에 있는 장벽을 없애는 플랫폼이다.

## 슬로건

> "Solve your problem with the right service."

## 핵심 신념

- 능력은 있는데 사업 운영 때문에 못 하는 사람은 없어야 한다
- 일자리를 잃는 사람은 Insty의 고객이 된다 — 도메인 지식은 있으니 실행만 도와주면 된다
- 쓰레기 비즈니스는 시장이 알아서 걸러낸다 — Insty는 심판이 아니라 출발선을 깔아주는 역할
- 기술은 수단이다 — AI가 아니라 "가치 실현"이 본질

## 로드맵

### Phase 1 — 대행 서비스 (Now ~ 4주)
```
손으로 세팅해주기 / 가이드 팔기
├── AI 세일즈 파이프라인 구축 대행
├── AI 에이전트 시스템 가이드 PDF
├── 개인 PC 직접 세팅 서비스
└── 수익: 시간 판매 (1회성)
```
**목적**: 첫 매출 + 도메인 지식 축적
- 어떤 업종에 어떤 에이전트가 필요한지 학습
- 어떤 워크플로우가 실제로 작동하는지 검증
- 고객 피드백으로 제품화 방향 결정

→ 상세: `phase1-plan.md`

### Phase 2 — 템플릿 마켓 (3~6개월)
```
업종별 AI 에이전트 "스타터킷"
├── 카페 사장님 킷: 재고관리 + SNS + 리뷰응대
├── 코치/컨설턴트 킷: CRM + 콘텐츠 + 세일즈
├── 쇼핑몰 킷: CS봇 + 상품설명 + 리뷰분석
├── 프리랜서 킷: 견적 + 포트폴리오 + 팔로업
└── 수익: 디지털 프로덕트 (무한 복제)
```
**목적**: 노동에서 제품으로 전환
- Phase 1에서 반복되는 세팅을 템플릿화
- 구매 → 원클릭 복제 → 바로 사용
- 크몽 Notion 템플릿의 "AI 에이전트 버전"

**전환 신호**: 같은 유형 세팅 요청 5건 이상 → 템플릿화

### Phase 3 — SaaS 플랫폼 (1~2년)
```
insty.ai — "1인 기업 AI 운영체제"
├── 가입 → 비즈니스 유형 선택
├── AI가 에이전트 팀 자동 구성
├── 대시보드에서 에이전트들 지시/모니터링
├── Slack/Notion/이메일 자동 연동
├── 수익: 월 구독 ₩9.9만~29.9만/월 (MRR)
└── 비개발자도 에이전트를 조립할 수 있는 UI
```
**목적**: 대행 → 셀프서비스 전환
- Phase 1~2에서 쌓은 도메인 지식이 플랫폼의 핵심 자산
- 핵심 차별화: 업종별 베스트 프랙티스가 내장된 플랫폼

**전환 신호**: 템플릿 구매자 100명+ & 반복 패턴 확립

### Phase 4 — AI COO (2~3년)
```
"사장님, 오늘 뭐 하실 건가요?"

AI가 알아서:
├── 어제 들어온 리드 3건 팔로업 완료
├── 이번 주 SNS 콘텐츠 5개 작성 대기
├── 매출 리포트: 지난주 대비 +12%
├── 추천: "이 고객에게 업셀 제안하세요"
└── 사장님은 승인 버튼만 누름
```
**목적**: 에이전트 "팀" → 에이전트 "COO"
- 사장의 역할: "일하는 사람" → "의사결정자"
- AI가 실행, 사람은 방향과 승인만

### Phase 5 — 1인 기업 네트워크 (3~5년)
```
Insty 위의 1인 기업들이 서로 연결

├── 디자이너 A의 AI팀 ←→ 개발자 B의 AI팀
│   → "A 고객이 앱 개발 필요" → B에게 자동 매칭
├── 코치 C ←→ 마케터 D
│   → AI끼리 협업 범위 조율 → 사람은 최종 승인만
└── 1인 기업 100개 = AI로 연결된 가상 대기업
```
**목적**: 개별 1인 기업이 AI를 통해 연합
- 프리랜서 마켓플레이스(크몽)의 완전 자동화 버전
- 수익: 거래 수수료 (네트워크 효과)

### Phase 6 — 비즈니스 자동 런칭 (5년+)
```
"나 이런 사업 하고 싶어"
→ AI가 사업계획, 법인설립, 브랜딩, 첫 고객 확보까지 실행
→ 사람은 비전만 제시
```

### Phase 7 — 가치 발견 플랫폼 (궁극의 형태)

기술이 극한까지 발전하면 실행 비용은 0이 된다.
누구나 무엇이든 할 수 있는 세상. 그때 남는 질문은 하나:

> **"나는 뭘 하고 싶은 거지?"**

```
기술이 해결하는 것:
  ✅ 어떻게 만들지 (실행)
  ✅ 어떻게 팔지 (마케팅)
  ✅ 어떻게 운영하지 (시스템)
  ✅ 돈이 없어서 못 하겠어 (자원)

기술이 해결 못 하는 것:
  ❌ 나는 뭘 하고 싶은가
  ❌ 이게 정말 내가 원하는 삶인가
  ❌ 내 가치는 뭔가
```

선택지가 무한해질수록 사람은 더 길을 잃는다.
Insty는 그 속에서 나침반이 된다.

```
Insty Phase 7:
├── 다양한 일을 시도해볼 수 있는 탐색의 장
├── 시도 → 피드백 → 발견의 사이클
├── "이런 사람들은 이런 일에서 가치를 찾았어요" 데이터
├── 커뮤니티: 비슷한 여정을 걷는 사람들의 연결
└── 궁극적으로: 자기 발견을 돕는 플랫폼
```

**도구는 대체되지만, "나를 찾는 여정"은 대체되지 않는다.**

## Insty가 의미없어지는 시나리오 & 방어

### 1. "사업" 개념 자체의 소멸
- 위협: 포스트-스캐시티 경제, UBI 세상
- 방어: Insty는 "사업 도구"가 아니라 "가치 실현 인프라" → 돈이 필요 없는 세상에서도 사람은 하고 싶은 일을 찾고 싶어함

### 2. 빅테크가 무료로 제공
- 위협: Google/Apple이 "비즈니스 자동화"를 OS에 내장
- 방어: 빅테크는 범용 도구, Insty는 업종별 맞춤 경험 (Shopify가 아마존 옆에서 생존한 논리)

### 3. "사업 시스템" 자체가 불필요
- 위협: 유튜브/인스타만으로 가치 실현 가능, 굳이 CRM 필요 없음
- 방어: **이것이 가장 본질적인 위험.** "사업 시스템 구축"에 갇히면 죽는다. "하고 싶은 일을 찾고 실현하는 플랫폼"으로 정체성 확장 필수 → Phase 7

### 결론
> 기술이 아무리 발전해도 "나는 뭘 하고 싶은가"는 본인만 답할 수 있다.
> Insty는 그 답을 찾는 여정을 돕는 한, 살아남는다.

| Phase | 수익 모델 | 스케일 | 마진 |
|---|---|---|---|
| 1. 대행 | 시간 판매 (1회성) | 월 수백만 | 높음 (노동) |
