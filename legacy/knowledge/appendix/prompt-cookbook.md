# 복붙 프롬프트 모음

Claude Code에서 바로 사용할 수 있는 프롬프트 모음입니다. `[대괄호]` 부분은 프로젝트에 맞게 교체하세요. Claude가 프로젝트를 스스로 분석하므로, 대괄호 부분을 삭제해도 대부분 잘 동작합니다.

## 1. CLAUDE.md 자동 생성

```
이 프로젝트의 전체 구조를 분석해서 CLAUDE.md를 작성해줘.
포함할 내용:
- 프로젝트 개요 (한 줄 설명)
- 기술 스택
- 디렉토리 구조와 각 디렉토리의 역할
- 주요 컨벤션 (네이밍, 에러 처리, 로깅)
- 테스트 실행 방법
- 빌드/배포 명령어
- 자주 쓰는 Claude Code Skills (/commit, /review 등)
```

## 2. 세션 메모리 정리

```
지금까지 작업한 내용을 정리해줘.
다음 세션에서 바로 이어서 작업할 수 있도록:

1. 완료된 작업 목록
2. 진행 중인 작업 (현재 상태)
3. 다음에 해야 할 작업
4. 중요한 결정사항과 이유
5. 발견한 이슈나 주의사항

docs/progress.md에 저장해줘.
```

## 3. 코드 리뷰 + 커밋

```
변경사항을 리뷰하고 커밋해줘:

1. git diff로 변경사항 확인
2. 잠재적 문제 체크:
   - 하드코딩된 값
   - 에러 핸들링 누락
   - 테스트 누락
3. 문제 없으면 Conventional Commits 형식으로 커밋
4. 문제 있으면 수정 후 커밋
```

## 4. 멀티파일 리팩토링

```
[대상 설명, 예: "인증 모듈을 리팩토링"]해줘.

단, 안전하게 진행하기 위해:
1. 먼저 영향받는 모든 파일을 파악해줘
2. 변경 계획을 보여줘 (파일별로)
3. 내가 승인하면 실행
4. 변경 후 테스트 실행
5. 테스트 통과하면 커밋
```

## 5. Hook 자동 설정

```
이 프로젝트에 맞는 Claude Code Hook을 설정해줘.
.claude/settings.json에 추가할 내용:

1. PreToolUse Hook:
   - 코드 수정 전 자동 포맷팅 (prettier/black)
   - 위험한 명령어 차단 (rm -rf, git push --force)

2. PostToolUse Hook:
   - 파일 수정 후 자동 린트
   - 테스트 자동 실행 (변경된 파일 관련)

3. Notification Hook:
   - 작업 완료 시 알림

[프로젝트 스택]에 맞게 구성해줘.
```

## 6. MCP 서버 추천 + 설치

```
이 프로젝트에 유용한 MCP 서버를 추천해줘.

프로젝트 정보:
- 기술 스택: [스택]
- 사용 중인 서비스: [GitHub, Notion, Slack 등]
- 주요 작업: [개발, 문서 작성, 프로젝트 관리 등]

각 MCP 서버에 대해:
1. 어떤 작업에 도움이 되는지
2. 설치 방법
3. .claude/settings.json 설정

추천 후 설치까지 진행해줘.
```

## 7. 프롬프트 개선

```
내가 자주 쓰는 프롬프트를 분석하고 개선해줘:

현재 프롬프트:
"""
[현재 사용 중인 프롬프트]
"""

개선 방향:
1. 더 구체적인 지시로 변환
2. 출력 형식 명시
3. 제약 조건 추가
4. few-shot 예시 포함
5. 개선된 프롬프트와 원본 비교
```

## 8. 테스트 자동 생성

```
이 프로젝트의 테스트 커버리지를 높여줘:

1. 테스트가 없는 함수/모듈 찾기
2. 우선순위 정하기:
   - 비즈니스 로직 > 유틸리티 > UI
   - 복잡도 높은 함수 우선
3. [테스트 프레임워크]로 테스트 작성
4. 엣지 케이스 포함
5. 테스트 실행 후 결과 보고
```

## 9. 반복 작업을 Skill로 만들기

```
내가 자주 하는 작업을 Claude Code Skill로 만들어줘:

작업 설명:
"""
[반복하는 작업 설명, 예: "PR 올리기 전에 린트, 테스트, 커밋 메시지 검사를 한번에"]
"""

.claude/skills/ 디렉토리에 SKILL.md로 저장해줘.
포함할 내용:
- YAML frontmatter (name, description)
- 단계별 실행 지침
- 필요한 도구 권한 (allowed-tools)
```

## 사용 팁

- CLAUDE.md부터 시작하세요. 다른 프롬프트의 효과가 크게 향상됩니다.
- `[대괄호]` 부분은 힌트일 뿐, 삭제해도 Claude가 프로젝트를 스스로 분석합니다.
- 좋은 결과가 나오면 Skill로 변환해 재사용하세요.
