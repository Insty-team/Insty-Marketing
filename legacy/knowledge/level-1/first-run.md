# 첫 번째 실행

> 💡 **이 챕터에서 배우는 것**: Claude Code 인터페이스 이해, 첫 대화 시작, 권한 모드 선택

## 전제 지식

[설치 가이드](/docs/level-1/installation)와 [API 키 설정](/docs/level-1/api-key-setup)을 완료한 상태여야 합니다.

---

## 실습 준비: 테스트 폴더 만들기

```bash
mkdir claude-test
cd claude-test
```

기존 프로젝트 폴더를 사용해도 됩니다.

---

## Claude Code 시작하기

```bash
claude
```

초기 실행 시 환영 메시지가 표시되며, 자연어로 명령을 입력할 수 있습니다.

---

## 인터페이스 구성

화면의 주요 요소:

- **상단**: 모델 정보 및 토큰 사용량
- **중간**: Claude의 응답 영역
- **하단**: 사용자 입력 프롬프트

---

## 실습 예제

### 실습 1: 기본 대화

기본 대화로 도구의 기능을 파악합니다.

### 실습 2: 파일 생성

파일 생성 시 권한 확인이 나타나면, `y` (한 번만), `always` (항상), 또는 `n` (거부) 선택.

### 실습 3: 코드 수정

기존 코드 수정 및 개선 요청.

---

## 권한 모드

1. **기본 모드**: 모든 작업 전에 확인 요청 (초보자 권장)
2. **Auto-approve 모드**: 모든 작업을 자동 실행하며, 신뢰할 수 있는 작업에만 사용. `Shift+Tab`으로 토글.
3. **Headless 모드**: 자동화 및 CI/CD 파이프라인용

---

## 주요 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl+C` | 응답 중단 |
| `Ctrl+L` | 화면 지우기 |
| `Shift+Tab` | 자동 승인 모드 토글 |
| `Ctrl+R` | 대화 초기화 |
| `↑` / `↓` | 이전 명령 탐색 |

---

## 슬래시 커맨드

| 커맨드 | 설명 |
|--------|------|
| `/help` | 도움말 |
| `/status` | 설정 및 모델 정보 |
| `/cost` | 세션 비용 확인 |
| `/exit` | 종료 |

> 💡 `/cost`로 확인하면 첫 세션은 보통 $0.01 미만입니다.

---

## 핵심 정리

- `claude` 명령으로 시작, 자연어로 대화
- 파일 수정 시 권한 확인 → 안전하게 작업
- 단축키와 슬래시 커맨드로 효율적 사용
- 권한 모드를 상황에 맞게 선택
