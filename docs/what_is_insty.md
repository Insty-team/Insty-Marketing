## Insty란?

**"Solve your problem with the right AI service."**

AI powered / AI native 1인 창업가(solo preneur)를 위한 **정보 공유 플랫폼 & 커뮤니티**.
사람들이 실무 문제에 맞는 AI 서비스를 찾고, 실전 사례를 통해 바로 적용할 수 있게 연결하는 서비스.

## 타겟 유저: Solo Builders / Independent Workers

- 1인 창업가 / 사이드 프로젝트 운영자
- 프리랜서 (디자이너, 마케터, 개발자)
- 작은 팀 리더 (2~5명)
- 콘텐츠 크리에이터 (수익화 초기)

**핵심 페인포인트**: "AI 써야 할 것 같은데, 어떤 걸 써야 하는지 모르겠고, 비교할 시간도 없음"

## 핵심 제공 가치

1. **문제 중심 탐색** — AI 툴 이름 몰라도 문제로 시작
2. **검증된 실전 콘텐츠** — 실제 겪은 문제 + 해결 과정
3. **콘텐츠 → 스레드 → Q&A 연결** — 소비 후 참여까지
4. **AI 서비스 선택 부담 제거** — 상황에 맞는 AI 바로 연결

## 수익 모델: Crowd-backed Request (공동 요청)

### 핵심 구조
- 1:1 요청 ❌ → **여러 Learner가 같은 요청에 크레딧을 모음**
- Creator는 **수요가 충분히 검증된 요청만** 골라서 제작
- 미성사 시 전액 환불 → Learner 리스크 0

### 크레딧 & 수익 분배
| 항목 | 값 |
|---|---|
| 요청 크레딧 단가 | ₩5,000 |
| 요청 수익 분배 | Creator 65% / Insty 35% |
| Plus 멤버십 | ₩39,900/월 |
| 구독 수익 분배 | Creator pool 70% / Insty 30% |

### 멤버십 구조
- **Pro (무료)**: 콘텐츠 조회 제한, 요청 참여 가능 (추가 크레딧 구매)
- **Plus (₩39,900/월)**: 커뮤니티 Q&A 접근 + 요청 크레딧 2개 포함

### Learner UX
1. 요청 풀에서 관심 있는 요청 발견
2. 1 크레딧 참여 (에스크로 보류)
3. 목표 미달 시 전액 환불 / 달성 시 Creator 제작

### Creator UX
1. 대시보드에서 크레딧 누적이 빠른 요청 확인
2. "제작 수락" → 요청 Locked → 데드라인 시작 (7일)
3. 제작 완료 → 수익 정산

## 비용 구조

- 초기 인력: 창업자 1명 (추가 인력 없음)

## 핵심 가설 (MVP 검증 필수)

> **최우선**: A1 + A2 + A3가 동시에 성립하는가?
> → "Learner가 실제로 돈을 내고 공동 요청에 참여하는가?"

- **A1**: Learner는 "지식"이 아니라 참여·질문·요청 권한에 돈을 낸다
- **A2**: 크레딧 1개(₩5,000)는 심리적 부담이 낮다
- **A3**: 공동 요청은 1:1보다 더 많이 참여된다
- **A4**: Creator는 수요 검증된 요청을 선호한다
- **A5**: Crowd-backed 구조가 Creator retention을 높인다

## 전략적 결정사항

### D1. 강의 판매 ❌ → 문제 중심 구조 ⭕
### D2. 1:1 요청 ❌ → Crowd-backed Request ⭕
### D3. 콘텐츠 접근 과금 ❌ → 행동 권한(게시/질문/요청) 과금 ⭕
### D4. 외부 Creator 먼저 ❌ → Founder as First Creator ⭕
### D5. 특정 산업 도메인 ❌ → "Insty 빌딩 과정 공개" ⭕
### D6. 플랫폼 수익 극대화 ❌ → Creator 수익 극대화 ⭕

## Founder as First Creator 전략

- Insty를 만드는 과정 자체를 콘텐츠로 공개
- 문제의 진짜성(authenticity) 최대화
- AI 활용이 과장 없이 자연스럽게 드러남
- 콘텐츠 포맷: 문제 상황 → 트레이드오프 → AI 활용 → 선택 → 결과
