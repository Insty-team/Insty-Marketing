# 6차 고민 — 도구/플랫폼 전략 재검토

> 이전 차수에서 발전/수정한 점: Round 5에서 커뮤니티 비즈니스의 핵심 조건을 분석했고, 현재 Insty BM(what_is_insty.md)과 BUSINESS_PLAN 사이의 괴리를 발견했다. 이번에는 BUSINESS_PLAN의 Phase 3~7을 비전과 현실 양쪽에서 해부한다.

## 핵심 질문
기존 BUSINESS_PLAN의 Phase 3~7(SaaS → 에이전트 네트워크 → 자동 런칭 플랫폼)이 현실적인가? 비전에 맞는가? 수정이 필요하면 어떻게?

## 생각 전개

### 1. Phase별 현실성 점검

#### Phase 3-4: SaaS 전환 (Month 10~24)

**계획:** 대행 고객 80명 중 50명을 SaaS로 전환, M24에 400명 도달

**현실 점검:**
```
긍정적:
+ "대행 → SaaS" 전환 자체는 검증된 패턴 (많은 에이전시가 이렇게 함)
+ 저가 모델(₩2.9~14.9만/월)은 이탈률이 낮을 수 있음
+ 고객이 이미 사용 중인 서비스의 자동화 → 전환 마찰 낮음

우려:
- SaaS 개발 리소스: 1인이 대행 + SaaS 개발을 동시에? → 현실적으로 불가능
- M10에 SaaS 출시는 너무 빠름 — 실제로는 M12~15가 현실적
- 400명 구독자 = CAC × 400 — 고객 확보 비용을 어디서 충당?
- "24시간 매니저" 수준의 SaaS는 기술적 난이도 매우 높음
```

**판정: 방향은 맞으나, 타임라인과 범위가 비현실적.**

필요한 수정:
1. SaaS의 범위를 대폭 축소: "카톡 챗봇 + 예약 리마인드 + 주간 리포트" 정도
2. 타임라인: M15~18에 MVP 출시가 현실적
3. 개발은 외주 또는 no-code 도구(Bubble, Softr 등) 활용

#### Phase 5: AI 에이전트 협업 네트워크 (Month 20~30)

**계획:** 네일샵 AI와 꽃집 AI가 자동으로 거래를 제안하는 네트워크

**현실 점검:**
```
긍정적:
+ 비전으로서는 매력적. "소상공인의 LinkedIn" 같은 가치
+ 네트워크 효과 = 가장 강한 moat (Round 2에서 확인)

우려 (치명적):
- 전제 조건: "우리 SaaS를 쓰는 가게가 수백 곳" → 이게 M20에 가능한가?
- "AI끼리 거래"는 현재 기술로 가능하지만 — 문제는 기술이 아님
- 핵심 문제: 네일샵 사장님이 "꽃집 AI와 자동 거래"를 원하는가?
  → 대부분의 소상공인은 이런 니즈 자체가 없음
  → 이것은 "기술적으로 가능하니까 만들자"의 전형적 함정
- 네트워크 효과의 bootstrap 문제: 초기에 거래 상대가 없으면 가치 없음
```

**판정: 비전은 아름답지만 M20~30에 실현 가능성 매우 낮음. 3년 로드맵에서 제외하거나, 5년+ 비전으로 옮겨야 함.**

#### Phase 6-7: 비즈니스 자동 런칭 + 자기발견 (Month 28~36+)

**계획:** "아이디어만 있으면 AI가 사업을 런칭해주는 플랫폼"

**현실 점검:**
```
이것은 비전이지, 3년 계획이 아니다.

- "시장 조사 AI → 브랜딩 AI → 운영 AI → 마케팅 AI" — 각각이 독립적인 스타트업 규모의 프로덕트
- 1인이 36개월 안에 이것을 만드는 것은 불가능
- 이것이 실현되려면: 팀 + 투자 + 기존 고객 기반 필요
- Phase 1~4의 성공이 전제 조건

이것은 "꿈"으로 남겨두되, 3년 계획에서는 제외해야 한다.
```

**판정: 5~10년 비전으로 재분류. 3년 계획에서 삭제.**

### 2. 두 개의 BM 문서 사이의 모순

현재 프로젝트에 두 개의 BM이 공존:

| | what_is_insty.md | BUSINESS_PLAN.md |
|---|---|---|
| 타겟 | 솔로프리너, 크리에이터 (글로벌) | 소상공인 (한국) |
| 핵심 가치 | AI 서비스 큐레이션 + 커뮤니티 | 소상공인 AI 운영 자동화 |
| 수익 모델 | Crowd-backed Request + 구독 | 세팅비 + 구독 + 성과보수 |
| 기술 | Next.js + Spring Boot + FastAPI | Claude Code + Zapier + Notion |
| 팀 | 7명이 만들었지만 정리 예정 | 1인 (Sam) |
| 시장 | 미국/글로벌 | 한국 |

**이것은 사실상 두 개의 다른 사업이다.**

Sam의 상황: "플랫폼은 보존하되 나중에 재활용"이라고 했다. 이것은 현명한 판단이다.

### 3. 비전과 현실의 정합: 무엇을 버리고 무엇을 살릴 것인가

**비전: "개인이 가치를 창출할 수 있는 시대, 그 개인들이 자생할 수 있게 돕는 것"**

이 비전에 맞는 것과 맞지 않는 것:

```
✅ 비전에 맞는 것:
- 소상공인 AI 도입 대행 → "개인(소상공인)이 가치를 창출하도록 돕는 것"
- Build in Public → "개인(Sam 자신)이 가치를 창출하는 과정 공개"
- 커뮤니티 → "개인들이 서로 돕고 자생하는 구조"
- 템플릿/디지털 상품 → "개인이 스스로 할 수 있게 해주는 도구"

❌ 비전과 거리가 먼 것:
- AI 에이전트 간 자동 거래 → 기술 중심, 고객 중심 아님
- 비즈니스 자동 런칭 플랫폼 → 가치 있지만 현재 실행 불가능
- Crowd-backed Request → 양면 시장 부트스트래핑 문제로 당장 불가능
```

### 4. 수정된 Phase 구조 제안

```
Phase 1 (Month 1~6): 대행 + 콘텐츠 [현금 흐름 + 자산 축적]
├── 한국: 소상공인/솔로프리너 AI 대행 (저가 다수)
├── 글로벌: Build in Public 콘텐츠 (영어)
└── 자산: 프로세스 레시피, 데이터, 사례, 템플릿

Phase 2 (Month 7~12): 구독 + 디지털 상품 [반복 수익]
├── 한국: 대행 고객 구독 전환 + 새 고객 구독 직접 가입
├── 글로벌: 템플릿/디지털 상품 판매 (Gumroad 등)
├── 커뮤니티: 고객 단톡방 → 비공식 커뮤니티 시작
└── 자산: 업종별 벤치마크, 콘텐츠 라이브러리

Phase 3 (Month 13~24): 도구화 + 커뮤니티 [스케일]
├── 한국: 반복 작업 도구화 (no-code SaaS 또는 경량 웹앱)
├── 글로벌: 유료 커뮤니티 (빌더스 클럽)
├── 한국: 업종별 사장님 모임 (오프라인 포함)
└── 수익: 구독 + 디지털 상품 + 커뮤니티

Phase 4 (Month 24~36): 플랫폼 기반 확장 [moat 강화]
├── SaaS 고객 400명+ → 데이터 해자 확보
├── 커뮤니티 → 소상공인 간 협업 (수동으로 시작)
├── Insty 플랫폼 재활용 가능성 검토
└── 수익: 월 2,000만원+ (구독 + 커뮤니티 + 디지털 상품)
```

**기존 대비 변경점:**
1. Phase 5(에이전트 네트워크)를 3년 계획에서 제외 → 5년+ 비전
2. Phase 6-7(자동 런칭)을 3년 계획에서 제외 → 10년+ 비전
3. SaaS 범위를 대폭 축소 — "24시간 매니저" → "반복 작업 자동화 도구"
4. 디지털 상품과 커뮤니티를 명시적 수익원으로 추가
5. 타겟의 우선순위 명확화: 한국 소상공인 우선, 글로벌 솔로프리너 병행

### 5. "AI는 도구일 뿐" 원칙과의 정합성

수정된 Phase 구조에서 AI의 역할:
- Phase 1: AI로 대행 효율을 높임 (도구)
- Phase 2: AI로 콘텐츠 생산 속도를 높임 (도구)
- Phase 3: AI를 내장한 도구를 고객에게 제공 (도구를 파는 것)
- Phase 4: AI 데이터 분석으로 인사이트 제공 (도구가 만드는 가치)

**모든 Phase에서 AI는 "어떻게(how)"이지 "무엇(what)"이 아니다.** Sam이 파는 것은 "결과"이고, AI는 그 결과를 만드는 수단이다.

### 6. 장원영 모델과의 비교

```
장원영:
  브랜드 = 장원영 본인
  배급 = 음악/영상/광고
  복리 = 팬덤 성장

Sam (수정된 계획):
  브랜드 = "0에서 시작해서 AI로 사업 만드는 사람" (Build in Public)
  배급 = 콘텐츠 + 디지털 상품 + SaaS (한계비용 0)
  복리 = 데이터 × 커뮤니티 × 브랜드 누적
```

**기존 BUSINESS_PLAN과의 차이:** 기존 계획은 "기술(에이전트 네트워크, 자동 런칭)"이 복리를 만들어주기를 기대했다. 수정된 계획은 **"사람(브랜드, 커뮤니티, 관계)"**이 복리를 만든다. 기술은 이 복리를 가속하는 도구일 뿐.

## 이전 차수 오류 수정

Round 4에서 "phase1-plan.md와 BUSINESS_PLAN의 타겟이 다르다"고 지적했는데, 이것은 "오류"가 아니라 **"전략적 혼란"**이었다. 두 문서가 다른 시기에 다른 생각으로 작성되었기 때문. 수정된 Phase 구조에서는 **한국 소상공인 우선, 글로벌 솔로프리너 콘텐츠 병행**으로 우선순위를 명확히 했다.

다만 phase1-plan.md의 "세일즈 파이프라인 구축 서비스" (한국 솔로프리너 대상)는 유효할 수 있다. 소상공인보다 솔로프리너가 세팅비 ₩50~150만을 낼 가능성이 높기 때문. 이것은 Round 7(3억 달성 경로)에서 시나리오로 비교해야 한다.
