# 4차 고민 — 무료 대행 → 돈 버는 대행 → 그 너머

> 이전 차수에서 발전/수정한 점: Round 3에서 "대행 1건 → 자산 3개"라는 규칙의 중요성을 확인했다. 대행은 "노동=돈"이지만, 올바른 방식으로 하면 moat의 원재료를 축적할 수 있다. 이번에는 대행을 "시작점"으로 만드는 구체적 메커니즘을 설계한다.

## 핵심 질문
대행이 끝이 아닌 시작점이 되려면? 대행에서 추출할 수 있는 "자산"은 무엇이고, 각각 어떻게 수익으로 전환되는가?

## 생각 전개

### 1. 무료 대행 → 유료 대행의 전환 메커니즘

현재 Phase 1 계획: 무료로 시작 → 포트폴리오 확보 → 유료 전환

**이 전환이 실패하는 전형적 패턴:**
```
무료로 시작 → "무료니까 해주세요" 문화 정착
→ 유료로 바꾸면 "예전엔 공짜였는데?" 이탈
→ 다시 무료로 돌아감 → 영원한 무료 루프
```

**이것을 피하는 방법:**

**전략 1: 무료의 범위를 처음부터 명확히 제한**
```
무료: "진단" (현재 상태 분석 + 개선 방안 제시)
유료: "실행" (실제 세팅 + 운영)

→ 무료 진단만으로도 고객에게 가치 제공
→ "직접 하기 귀찮으시면 ₩50만에 해드릴게요"
→ 진단이 곧 세일즈 도구
```

**전략 2: 시간 제한 + 수량 제한**
```
"첫 3개 업체 무료 세팅" — 명확한 수량 제한
→ 4번째부터는 당연히 유료
→ 무료 3건의 사례가 곧 포트폴리오
```

**전략 3: 성과 연동**
```
세팅은 무료, 성과가 나면 수익 공유 (10~20%)
→ 고객 리스크 0 → 진입 허들 최소화
→ Sam의 리스크: 성과가 안 나면 공짜 일한 것
→ 하지만 사례와 데이터는 남음
```

**현실적 추천: 전략 1 + 2 조합**
- 첫 3건: 무료 진단 + 무료 실행 (포트폴리오용, 후기 필수)
- 4~10건: 무료 진단 + 유료 실행 (₩30~50만)
- 11건~: 유료 진단(₩10만) + 유료 실행 (₩50~100만)

### 2. 대행에서 추출하는 자산의 전체 맵

대행 1건에서 뽑아낼 수 있는 자산을 7개 카테고리로 분류:

#### 자산 1: 프로세스 레시피 (재사용 가능한 워크플로우)

| 예시 | 재사용 방법 | 수익화 |
|------|-----------|--------|
| 네일샵 예약 챗봇 세팅 과정 | 다음 네일샵은 30% 시간으로 세팅 | 효율 → 마진 증가 |
| 노쇼 방지 자동 리마인드 워크플로우 | 템플릿화하여 SaaS에 내장 | Phase 3 SaaS의 핵심 기능 |
| 인스타 콘텐츠 자동화 Zapier 레시피 | 다른 업종에도 변형 적용 | 디지털 상품으로 판매 가능 |

**축적 규칙: 대행 완료 후 24시간 내에 `/workspace/shared/insights/setup-patterns/`에 기록**

#### 자산 2: 업종별 벤치마크 데이터

| 데이터 | 가치 | 수익화 |
|--------|------|--------|
| 네일샵 평균 노쇼율 (세팅 전 vs 후) | Before/After 증거 = 세일즈 도구 | 후기/사례로 신규 고객 확보 |
| 업종별 최적 예약 리마인드 시간 | 다른 고객에게 즉시 적용 = 차별화 | SaaS 기본값으로 사용 |
| 숏폼 콘텐츠 타입별 전환율 | 콘텐츠 전략 최적화 | 컨설팅 가격 근거 |

**10건 이상 쌓이면: "네일샵 AI 도입 리포트" → 무료 배포 → 리드 자석**

#### 자산 3: 콘텐츠 소재

대행 1건에서 최소 5개 콘텐츠 추출 가능:

```
1. [숏폼] Before/After 비교 영상 (30초)
2. [블로그] "네일샵 A의 노쇼를 50% 줄인 방법" 사례 글
3. [X/LinkedIn] "오늘 배운 것" 빌드인퍼블릭 포스트
4. [뉴스레터] 주간 진행 리포트
5. [커뮤니티] "이런 문제 있으신 분?" 정보성 글
```

**핵심: 콘텐츠는 한 번 만들면 무한히 일한다 (한계비용 0 배급)**

#### 자산 4: 고객 관계 + 레퍼럴 네트워크

```
만족한 고객 A → "사장님, 아시는 분 중에 같은 고민 있으신 분?"
→ 소개 1건 → 세팅 → 또 소개
→ 네일샵 사장님들은 서로 알고 지냄 → 업종 내 입소문
```

**이것이 커뮤니티의 씨앗이 됨 (Phase 5로의 자연스러운 연결)**

#### 자산 5: 자동화 코드/도구

대행하면서 만드는 스크립트와 도구:

```
Python 스크립트: 예약 데이터 분석기
Zapier Zap: 예약 → 리마인드 → 노쇼 감지 → 블랙리스트
Notion 템플릿: 고객 관리 CRM
프롬프트: 업종별 콘텐츠 자동 생성
```

→ 이것들이 Phase 3-4 SaaS의 핵심 모듈이 됨
→ "코드를 짜는 시간"이 아니라 "제품의 MVP를 만드는 시간"

#### 자산 6: 고객의 목소리 (Voice of Customer)

```
"사장님, 이거 말고 또 필요하신 거 있으세요?"
→ 대부분의 요구사항은 비슷할 것 (가설)
→ 공통 요구사항 = 다음 제품의 기능 정의
→ PM이 할 고객 인터뷰를 대행하면서 자연스럽게 수행
```

#### 자산 7: Sam의 경험과 판단력

가장 눈에 안 보이지만 가장 중요한 자산:

```
10건 대행 → "이 업종은 이렇게 하면 된다" 패턴 인식
50건 대행 → "이 문제는 이런 식으로 접근해야 한다" 전문성
100건 대행 → "무엇이 되고 무엇이 안 되는지 안다" 시장 통찰
```

→ 이것이 컨설팅/멘토링으로 수익화될 수 있음
→ "해본 사람의 조언"은 AI가 대체할 수 없음

### 4. "그 너머" — 대행 졸업 후의 세계

대행에서 졸업하는 시점의 조건:

```
조건 1: 구독 MRR이 대행 수입을 넘어설 때 (대행 의존도 감소)
조건 2: 템플릿/디지털 상품의 반복 매출이 생활비를 커버할 때
조건 3: SaaS MVP가 준비되어 신규 고객을 자동 온보딩할 수 있을 때
```

**졸업 후 Sam의 역할 변화:**

| 시기 | 역할 | 시간 배분 |
|------|------|----------|
| Month 1~6 | 실행자 (직접 세팅) | 대행 70% / 콘텐츠 30% |
| Month 7~12 | 실행자 + 설계자 | 대행 50% / SaaS 설계 30% / 콘텐츠 20% |
| Month 13~18 | 설계자 + 콘텐츠 크리에이터 | SaaS 운영 40% / 콘텐츠 40% / 대행 20% |
| Month 19~24 | 크리에이터 + 커뮤니티 운영자 | 콘텐츠 40% / 커뮤니티 30% / SaaS 30% |

**핵심: 대행에서 손을 떼는 것이 아니라, 대행의 비중을 자산 수익이 대체하도록 만드는 것**

### 5. 현재 BUSINESS_PLAN과의 간극

BUSINESS_PLAN에서는 Phase 1-2(대행)와 Phase 3-4(SaaS) 사이에 명확한 "자산 전환 메커니즘"이 빠져 있다.

**빠진 것:**
1. 대행 → 자산 추출 프로세스 (자동화되어야 함)
2. 디지털 상품 라인 (템플릿, 가이드) — 현재 "무료 미끼"로만 사용
3. 콘텐츠 수익화 채널 (YouTube AdSense, 후원 등)
4. 데이터 자산의 구체적 활용 계획

**이것은 Round 8(90일 계획)에서 구체적으로 보완해야 함.**

### 6. Phase 1 계획(phase1-plan.md)과의 정합성

현재 phase1-plan.md는 "AI 세일즈 파이프라인 구축"을 서비스로 정의하고 있다. 이것은 BUSINESS_PLAN의 소상공인 타겟(네일샵, 베이커리)과 다르다.

**두 개의 타겟이 존재:**
- phase1-plan.md: 한국 솔로프리너/프리랜서 (세일즈 파이프라인)
- BUSINESS_PLAN: 소상공인 (네일샵, 베이커리, 카페)

**이 두 트랙을 동시에 할 수 있는가?** 1인이 두 개의 타겟을 동시에 공략하면 둘 다 어중간해질 리스크가 있다. Round 8에서 하나를 선택하거나, 순서를 정해야 한다.

## 이전 차수 오류 수정

Round 3에서 "대행 1건에서 최소 3개의 자산을 뽑아내는 습관"이라고 했는데, 분석해보니 **7개 카테고리의 자산**을 뽑아낼 수 있다. 3개는 과소평가였다. 다만, 7개를 모두 의식적으로 추출하려면 시스템이 필요하다 — 대행 완료 체크리스트를 만들어야 한다.

또한, Round 2에서 BUSINESS_PLAN의 Phase 1-2가 "가장 취약하다"고 했는데, 정확히는 **Phase 1-2의 "대행 행위" 자체가 취약한 것이 아니라, "대행만 하고 자산을 안 쌓는 것"이 취약한 것**이다. 대행은 올바르게 수행하면 가장 빠른 moat 구축 수단이 된다.
