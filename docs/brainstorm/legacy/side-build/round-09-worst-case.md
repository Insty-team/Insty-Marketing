> 핵심 질문: "실패하면 어떻게 되는가? 어떻게 대응하는가?"

## 1. 사이드 빌딩의 실패 확률 (현실 직시)

### 인디 해커 실패 통계

| 시도 | 성공 확률 | 출처 |
|------|---------|------|
| 단일 제품 MRR $1,000 도달 | 10-15% | Indie Hackers 커뮤니티 평균 |
| 3개 제품 중 1개 MRR $1,000 | 27-39% | 확률 계산 |
| 5개 제품 중 1개 MRR $1,000 | 41-56% | 확률 계산 |
| 10개 제품 중 1개 MRR $1,000 | 65-80% | Pieter Levels 전략 |

> **핵심**: 개별 제품은 실패할 확률이 높다. 하지만 여러 번 시도하면 확률이 올라간다.
> 4년간 26개 프로젝트로 $115K를 번 인디 해커 사례가 이를 증명한다.

### Sam의 실패 리스크 분석

| 리스크 | 결과 | 손실 |
|--------|------|------|
| HookGen 실패 | 3개월 시간 투자 | 시간만 (금전 손실 거의 0) |
| 1년간 제품 3개 실패 | 12개월 × 주 18시간 | 약 900시간 (금전 손실 거의 0) |
| 2년간 MRR 0 | 24개월 투자 | 1,800시간 + 기회비용 |
| 정규직 해고 | 별도 리스크 | 사이드 안 해도 발생 가능 |

**최대 손실 = 시간.** 금전적 손실은 거의 없다 (도메인 비용, 서버 비용 = 월 $20-50).

## 2. 시나리오별 판단 기준과 대응

### Scenario A: 3개월 후 HookGen 매출 0

**상황**: 훅 DB도 안 팔리고, SaaS 가입자도 0명.

**판단 기준**:
- 트래픽은 있는데 전환이 0? → 가격/포지셔닝 문제
- 트래픽 자체가 0? → 마케팅/채널 문제
- 관심 자체가 없음? → 제품 아이디어 문제

**대응**:

| 원인 | 액션 |
|------|------|
| 가격 문제 | 무료 체험 추가, 가격 50% 인하, 프리미엄 기능 분리 |
| 마케팅 문제 | 트위터 전략 변경, Product Hunt 론칭, 레딧 마케팅 |
| 아이디어 문제 | **피봇**: NicheNewsAI 또는 ShipKit으로 전환 |

**피봇 결정 기준**: 50명 이메일 구독 또는 5건 프리세일을 2주 내 못 얻으면 → 피봇.

### Scenario B: 6개월 후 MRR $50 미만

**상황**: 제품은 만들었고 소수 유저가 있지만, 유료 전환이 안 됨.

**판단 기준**:
- 무료 유저가 많으면 → 유료 전환 유도 강화 (기능 제한, 프리미엄 유인)
- 무료 유저도 적으면 → 니치가 잘못됨
- 피드백에서 "이거 필요한데 XX가 부족해" → 기능 추가 후 재시도

**대응**:

```
Option 1: 니치 피봇 (같은 제품, 다른 시장)
  예: HookGen을 "크리에이터 전체"에서 "부동산 에이전트"로 좁힘

Option 2: 제품 피봇 (같은 시장, 다른 제품)
  예: 크리에이터 대상이지만 훅 대신 "캡션 생성기"

Option 3: 완전 전환 (다른 제품, 다른 시장)
  예: Round 3의 다른 아이디어로 이동
```

### Scenario C: 1년 후 총 수입 30만원

**상황**: 여러 시도를 했지만 의미 있는 수입이 없음.

**자기 점검 질문**:
1. 실제로 출시를 했는가? (빌드만 하고 론칭을 안 한 것 아닌가?)
2. 마케팅을 했는가? (좋은 제품을 만들고 아무도 모르는 것 아닌가?)
3. 고객과 대화했는가? (피드백 없이 추측으로 만든 것 아닌가?)
4. 몇 개의 제품을 시도했는가? (1개만 했으면 표본 부족)

**대응**:

| 문제 | 액션 |
|------|------|
| 론칭을 안 함 | "2주 안에 뭐든 출시" 챌린지 |
| 마케팅 부족 | 빌드 50% + 마케팅 50% 비율로 전환 |
| 고객 대화 없음 | 타겟 50명에게 DM 보내기 |
| 시도 횟수 부족 | Marc Lou 전략: 월 1개 제품 × 3개월 |

### Scenario D: 2년 후 MRR $50만원 (월)

**상황**: 나쁘진 않지만 기대에 미치지 못함.

**이건 실패가 아니다.**
- 월 50만원 × 12개월 = 연 600만원 추가 수입
- 정규직 + 50만원 = 월 495만원 → 저축 210만원 가능
- 3년 저축 = 7,560만원 (기존 6,060만 대비 +1,500만원)

**판단**:
- 성장 추세가 있는가? (50 → 80 → 120만?) → 계속
- 정체되어 있는가? → 피봇 또는 두 번째 제품에 집중
- 하락 중인가? → 근본 원인 분석 (경쟁자? 시장 변화?)

## 3. Plan B 카드들

### Plan B-1: Productized Service 전환

SaaS가 안 되면 같은 스킬로 대행 서비스:

| 서비스 | 가격 | 월 가능 건수 | 월 수입 |
|--------|------|------------|--------|
| AI 챗봇 설치 | $499 | 4건 | $2,000 |
| 세일즈 파이프라인 구축 | $999 | 2건 | $2,000 |
| 콘텐츠 자동화 시스템 구축 | $1,499 | 1건 | $1,500 |

**장점**: 즉시 수입 가능
**단점**: 시간=돈. 100억 경로 아님. 하지만 "3년 내 1억" 목표에는 도움

### Plan B-2: 디지털 상품 집중

SaaS 포기하고 디지털 상품 포트폴리오에 올인:

| 전략 | 예상 월 수입 |
|------|-----------|
| 상품 10개 × 평균 월 30만원 | 300만원 |
| 상품 20개 × 평균 월 20만원 | 400만원 |
| 히트 상품 1개 + 나머지 | 500만원+ |

**장점**: 빌드 빠름, 패시브 수입
**단점**: 100억 경로 아님. 하지만 "3년 내 1억"에 충분

### Plan B-3: AI 프리랜싱 (시급 높은 경로)

| 플랫폼 | 시급 | 주 10시간 | 월 수입 |
|--------|------|---------|--------|
| Toptal | $80-150 | 40-60h | $3,200-6,000 |
| Upwork (상위) | $50-100 | 40h | $2,000-4,000 |
| 직접 영업 | $100-200 | 20h | $2,000-4,000 |

**장점**: 가장 확실한 수입
**단점**: 이중 노동, 번아웃, 100억 경로 아님

### Plan B-4: 정규직 최적화 (사이드 포기)

최악의 경우, 사이드를 접고 정규직에 집중:
- SLAM 연봉 인상 (445 → 550 → 700만원 3년 목표)
- 또는 AI/ML 직군으로 이직 (연봉 점프)
- 월 저축 200만원 × 36개월 = 7,200만원

**이건 "실패"가 아니다.** 사이드 빌딩 과정에서 얻은 스킬은 연봉 협상/이직에 활용 가능.

## 4. 실패해도 괜찮은 구조

### 정규직이 안전망인 이유

```
[정규직 월급 445만원]
         ↓
[최소 생활비 + 대출 상환]
         ↓
[월 160만원 저축] ← 사이드 성공 여부와 무관하게 보장
         ↓
[3년 후 최소 6,000만원] ← 실패해도 이건 남음
```

### 사이드 빌딩 실패 시 남는 것

| 자산 유형 | 내용 | 가치 |
|----------|------|------|
| 기술 스킬 | AI 에이전트, 풀스택, 배포, 마케팅 | 이직/연봉 협상에 활용 |
| 포트폴리오 | 만든 제품들 (실패해도 포트폴리오) | GitHub/이력서 강화 |
| 네트워크 | 인디 해커 커뮤니티 인맥 | 다음 기회에 활용 |
| 시장 이해 | 어떤 것이 되고 안 되는지 체감 | 다음 시도에 확률 ↑ |
| 마케팅 스킬 | 트위터, SEO, Product Hunt | 범용 스킬 |

> **"실패"의 정의를 바꿔라**: 돈을 못 번 것이 아니라, **돈을 벌 방법을 배운 것**이다.

## 5. 피봇 타이밍 체크리스트

### 매 분기 자문해야 할 질문

```markdown
## 분기 리뷰 — [날짜]

### 숫자
- 이번 분기 MRR: $__
- 지난 분기 대비 성장률: __%
- 유료 고객 수: __
- 이번 분기 총 수입 (디지털 상품 포함): __만원

### 판단
- [ ] 성장 추세가 보이는가? → 계속
- [ ] 정체되어 있는가? → 원인 분석 후 3주 집중 개선
- [ ] 하락 중인가? → 피봇 검토
- [ ] 고객이 추천해주는가? → 되면 PMF 확인

### 감정
- [ ] 아직 즐거운가? → 계속
- [ ] 번아웃 느끼는가? → 1주 완전 휴식 후 재평가
- [ ] 의무감으로만 하는가? → 진지하게 피봇/중단 검토

### 결정
- [ ] 유지
- [ ] 니치 피봇
- [ ] 제품 피봇
- [ ] 완전 전환
- [ ] 일시 중단 (1개월)
```

## 6. 최악의 최악: 모든 것이 실패한 경우

### 3년 동안 사이드 빌딩 + MRR 0원이면?

**남는 것**:
- 정규직 저축 6,060만원 (= 사이드 안 했어도 같은 금액)
- AI/풀스택/마케팅 스킬 (= 더 높은 연봉으로 이직 가능)
- 인디 해커 네트워크
- "시도해 본" 경험 (= 후회 없음)

**잃은 것**:
- 3년 × 주 18시간 = 약 2,800시간
- 일부 연인/사회 시간

**결론**: 최악의 경우에도 **"시간 낭비"가 최대 손실**이다. 그리고 그 시간도 스킬 향상에 쓰였으므로 순수 낭비는 아니다.

### 비교: 사이드 안 한 3년 vs 사이드 한 3년

| 항목 | 사이드 안 함 | 사이드 함 (실패) | 사이드 함 (성공) |
|------|-----------|---------------|---------------|
| 저축 | 6,060만 | 6,060만 | 1.2-1.4억 |
| 스킬 | SLAM만 | SLAM + AI/풀스택/마케팅 | 동일 + 사업 스킬 |
| 이직 옵션 | 제한적 | 넓어짐 | 필요 없음 |
| 후회 | "해볼 걸" | 없음 | 없음 |

## 7. Round 9 결론

### 핵심 메시지

**"정규직이 안전망이니까, 리스크는 '시간 투자'뿐이다. 실패해도 괜찮다."**

1. **개별 제품 실패 확률 높다 (85-90%)**: 하지만 여러 번 시도하면 확률 올라감
2. **피봇은 실패가 아니다**: 방향 수정일 뿐. 3개월마다 판단
3. **최악의 경우에도 남는 것이 있다**: 스킬, 포트폴리오, 경험
4. **Plan B가 여러 개 있다**: 대행, 디지털 상품, 프리랜싱, 정규직 최적화
5. **"3년 내 1억"은 정규직만으로도 60% 달성**: 사이드는 나머지 40%를 채우는 것

### Sam에게의 메시지

> HookGen이 실패하면 NicheNewsAI를 만들면 된다. 그것도 실패하면 ShipKit을 만들면 된다. 3개 다 실패하면 디지털 상품 10개를 만들면 된다. 그것도 안 되면 AI 프리랜싱을 하면 된다. **옵션은 무궁무진하고, 최악의 경우에도 잃는 것은 시간뿐이다.**
>
> 가장 큰 실패는 "아무것도 시도하지 않는 것"이다.
