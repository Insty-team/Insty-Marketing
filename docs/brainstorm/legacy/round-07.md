# 7차 고민 — 3억 달성 경로

> 이전 차수에서 발전/수정한 점: Round 6에서 BUSINESS_PLAN의 Phase 3~7을 현실 점검하고, Phase 구조를 수정했다. 에이전트 네트워크와 자동 런칭 플랫폼은 3년 계획에서 제외하고, 대행→구독→도구화→플랫폼 확장의 4단계로 단순화했다. 이번에는 이 수정된 구조로 진짜 3억이 모이는지 숫자로 검증한다.

## 핵심 질문
Round 1~6을 종합해서, 현실적으로 3년 안에 3억 모을 수 있는 경로는? 복수 시나리오로 검증.

### 5. 추천 경로

**시나리오 B(솔로프리너 중심)를 기본으로, 시나리오 A의 요소를 선별 차용.**

이유:
1. **고단가 = 더 적은 고객으로 같은 수익** → 1인 운영에 적합
2. **솔로프리너 = 온라인 접근 가능** → 물리적 영업 불필요
3. **Build in Public = 브랜드 구축과 세일즈가 동시에 진행** → 효율적
4. **글로벌 콘텐츠 = 달러 수입** → 환율 효과 + 시장 규모

**시나리오 A에서 차용할 것:**
- 소상공인 대행도 기회가 오면 거절하지 않음 (수익원 다변화)
- 업종별 데이터 축적은 유지 (장기적으로 가치)
- 저가 구독 모델의 장점(낮은 이탈률)은 SaaS 설계에 반영

### 6. 3억 달성을 위한 핵심 마일스톤

```
Month 3:  유료 고객 3건 이상, 월 수입 ₩200만+ (생존 확인)
Month 6:  유지보수 고객 5명+, 디지털 상품 출시 (반복 수익 시작)
Month 9:  월 수입 ₩500만+ (생활비 + 저축 가능)
Month 12: 월 수입 ₩800만+ (목표선 진입)
Month 18: 월 수입 ₩1,200만+ (목표선 안정)
Month 24: 누적 저축 ₩1.0억+ (중간 점검)
Month 30: 누적 저축 ₩2.0억+ (매물 탐색 시작)
Month 36: 누적 저축 ₩3.0억+ (계약 가능)
```

**Month 3이 가장 중요한 게이트.** 유료 고객 3건을 못 만들면 전체 계획 재검토 필요.

## 이전 차수 오류 수정

기존 BUSINESS_PLAN의 시뮬레이션(Section 7.2)에서 "보수적 달성률 50% 적용 → 3.12억"이라고 했는데, 이 시뮬레이션은 Phase 5(에이전트 네트워크)의 M25~30 월수입 3,500만원을 포함한 것이다. Phase 5를 제외하면 이 수치는 달성 불가능하다.

**수정된 시뮬레이션에서는 Phase 5 없이도 3억 달성 가능한 경로를 설계했다.** 핵심은 고단가 솔로프리너 서비스 + 디지털 상품 + 커뮤니티의 조합.
