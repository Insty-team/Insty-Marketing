# 2차 고민 — 기술 발전에 흔들리지 않는 비즈니스란?

> 이전 차수에서 발전/수정한 점: Round 1에서 "개인 가치 창출 시대"가 실재하는 트렌드임을 확인했다. 하지만 "돕는 것"이 기술 발전으로 가치가 사라질 수 있다는 리스크를 발견했다. 이번에는 이 리스크를 정면으로 다룬다.

## 핵심 질문
AI가 5년, 10년 후 어떻게 변해도 살아남는 비즈니스의 특성은 무엇인가? Sam의 비즈니스는 그 특성을 갖추고 있는가?

## 생각 전개

### 1. 역사적 사례: 기술 변화에 죽은 것과 산 것

**죽은 것들의 공통점:**

| 회사 | 기술 변화 | 사인(死因) |
|------|----------|-----------|
| Kodak | 디지털 카메라 | 자기가 발명한 기술을 두려워해 외면 |
| Blockbuster | 스트리밍 | Netflix의 $50M 인수 제안을 거절 (2000년) |
| Nokia | 스마트폰 | "우리 시장점유율"에 안주, 소프트웨어 생태계를 무시 |
| 여행사 대부분 | 인터넷 | "정보 중개" 자체가 가치였는데, 정보가 무료가 됨 |
| 타이피스트 | 워드프로세서 | "기능(타이핑 스킬)" 자체가 상품이었는데, 기능이 자동화됨 |

**살아남은 것들의 공통점:**

| 회사/업종 | 기술 변화 | 생존 이유 |
|----------|----------|----------|
| Netflix | DVD→스트리밍 | **스스로를 파괴**. DVD가 수익적일 때 스트리밍으로 전환 |
| 어도비 | 패키지→클라우드 | 라이선스 판매→구독(SaaS)으로 자기 모델을 뒤집음 |
| 고급 여행사 | 인터넷 | "정보"가 아니라 "큐레이션+경험+네트워크"로 가치 재정의 |
| 회계사 | 엑셀/회계 소프트웨어 | "계산"이 아니라 "판단+신뢰+규제 대응"으로 가치 이동 |
| 미용사/셰프 | YouTube 튜토리얼 | Physical 실행은 대체 불가. 정보는 무료여도 손기술은 못 복사 |

### 2. 패턴 추출: 기술에 죽는 것 vs 사는 것

**죽는 비즈니스의 특성:**
1. **정보 중개가 핵심 가치** — 정보가 무료가 되면 끝
2. **기능(skill) 자체가 상품** — 기능이 자동화되면 끝
3. **가치사슬의 중간에만 위치** — 양쪽이 직접 연결되면 끝
4. **전환비용(switching cost)이 낮음** — 더 나은 대안이 나오면 즉시 이탈

**사는 비즈니스의 특성:**
1. **네트워크 효과** — 사용자가 많을수록 가치가 올라감 (LinkedIn, Airbnb)
2. **브랜드/신뢰** — 기술이 바뀌어도 "이 사람/회사를 믿는다"는 감정은 잔존
3. **워크플로우 통합** — 고객의 일상에 깊이 박히면 교체 비용이 높아짐
4. **데이터 해자(moat)** — 축적된 독점 데이터가 있으면 AI 시대에 오히려 강해짐
5. **Physical 실행** — 디지털로 대체할 수 없는 물리적 행위

### 3. Sam의 현재 계획을 이 프레임으로 진단

**현재 BUSINESS_PLAN의 각 Phase를 살아남는 특성과 대조:**

| Phase | 핵심 가치 | 죽는 특성? | 사는 특성? |
|-------|----------|-----------|-----------|
| Phase 1-2: AI 대행 | "내가 세팅해줄게" (기능 판매) | **위험** — AI가 쉬워지면 세팅 자체가 불필요해짐 | 약함 — 전환비용 낮음, 네트워크 없음 |
| Phase 3-4: SaaS | "셀프서비스 도구" | **중간** — 더 나은 SaaS가 나오면 이탈 가능 | 워크플로우 통합으로 전환비용 확보 가능 |
| Phase 5: 에이전트 네트워크 | "AI끼리 거래" | 낮음 | **강함** — 네트워크 효과 발동 |
| Phase 6-7: 자동 런칭 | "아이디어→사업" | 중간 | 데이터 해자 + 네트워크 가능 |

**문제: Phase 1-2가 가장 취약하다.**

AI 대행은 본질적으로 "기능 판매"다. "노쇼 방지 문자 세팅", "인스타 자동 포스팅 세팅" — 이것들은 1년 후면 Zapier나 ChatGPT가 "사장님, 그냥 이렇게 하세요"라고 알려줄 수 있는 것들이다.

### 4. 기술 변화에 흔들리지 않는 비즈니스의 "진짜 moat"

Warren Buffett의 moat 프레임워크를 AI 시대에 재해석:

**1) 네트워크 효과 (가장 강력)**
- GenAI 시대에도 네트워크 효과는 여전히 유효
- 단, "사용자 수 = 가치"인 단순 네트워크는 약해지고 있음
- **"데이터 + 관계 + 거래"가 얽힌 복합 네트워크**가 진짜 moat
- Sam에게의 시사: 소상공인들의 "관계 네트워크"를 만들면 이것은 AI로 대체 불가

**2) 전환비용 (중요하지만 불충분)**
- SaaS의 per-seat 모델이 AI로 위협받고 있음 — 1명이 5명 일을 하면 seat이 줄어듦
- **워크플로우 깊숙이 파고든 도구**만 전환비용이 높음
- Sam에게의 시사: "세팅해주고 끝"이 아니라 "매일 쓰는 도구"가 되어야 함

**3) 브랜드/신뢰 (느리지만 견고)**
- 기술이 바뀌어도 "이 사람 말이면 믿을 수 있다"는 남음
- Coinbase가 암호화폐 시장에서 신뢰로 moat를 구축한 것과 같음
- Sam에게의 시사: **Sam 개인의 브랜드**가 장기적으로 가장 강한 자산이 될 수 있음

**4) 독점 데이터 (AI 시대의 핵심)**
- AI 모델은 범용. 하지만 **특정 도메인의 독점 데이터**를 가진 자가 승리
- 네일샵 100곳의 노쇼 패턴 데이터, 예약 시간대 데이터 — 이런 것은 OpenAI도 없음
- Sam에게의 시사: 대행하면서 데이터를 체계적으로 축적하면, 이것이 Phase 3-4의 핵심 자산

### 5. "AI는 도구일 뿐" 원칙의 재검토

Sam의 원칙 "AI는 도구일 뿐"은 옳다. 하지만 현재 BUSINESS_PLAN은 이 원칙과 미묘하게 모순된다:

- "AI 대행 서비스" — 이름 자체가 AI를 앞세움
- "AI 세일즈 파이프라인 구축" — AI가 핵심 selling point
- 금지 용어로 "AI OS, SaaS, 에이전트"를 지정했지만 — 서비스 이름 자체는 "AI"를 포함

**진짜 "AI는 도구일 뿐"이라면:**
- 파는 것은 "결과"여야 함: "매출 20% 올려드립니다", "노쇼 50% 줄여드립니다"
- AI는 그 결과를 달성하는 수단일 뿐
- 기술이 바뀌면 수단만 바꾸면 됨 — 가치(결과)는 동일

### 6. 결론: Sam이 만들어야 할 moat의 순서

```
지금 (Month 1~6):    브랜드/신뢰 구축 ← 대행하면서 사례와 콘텐츠 축적
                     + 독점 데이터 축적 ← 업종별 패턴 기록

중기 (Month 7~18):   워크플로우 통합 ← 매일 쓰는 도구 되기
                     + 전환비용 확보

장기 (Month 18~36):  네트워크 효과 ← 소상공인 간 연결
                     + 복합 moat (데이터 × 네트워크 × 브랜드)
```

핵심 교훈: **대행(Phase 1)은 moat가 아니라 moat를 쌓기 위한 "삽질"이다.** 삽질 자체가 목적이 되면 안 되고, 삽질하면서 쌓이는 것(데이터, 브랜드, 관계)이 진짜 자산이다.

## 이전 차수 오류 수정

Round 1에서 "트렌드는 실재하나, Sam에게 중요한 건 다른 질문"이라고 했는데, 이를 더 정확히 하면:

- 트렌드가 실재한다 해도, **그 트렌드 위에서 "moat 없는 비즈니스"를 하면 의미 없다**
- 비전이 맞는 것과 비즈니스가 되는 것은 별개라고 했는데, 정확히는 **비전이 맞아도 moat가 없으면 경쟁에서 밀린다**는 것

## 다음 차수에서 더 파야 할 것
1. Sam의 현실(매출 0, 인지도 0)에서 "브랜드 구축"과 "돈 벌기"를 동시에 할 수 있는 경로는?
2. 장원영 모델(대체불가 브랜드 × 한계비용 0 배급 × 복리)을 Sam의 상황에 구체적으로 적용하면?
3. "노동=돈" 공식을 깨는 현실적 방법 — Phase 1의 대행은 완전히 "노동=돈" 구조인데, 여기서 어떻게 빠져나오는가?
