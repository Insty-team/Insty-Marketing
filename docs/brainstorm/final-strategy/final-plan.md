## 2. 운영 원칙: 데이터 기반 가설-검증 사이클

모든 의사결정을 **"데이터 수집 → 초기 가설 → 실험 → 시장 검증 → 판단"** 사이클로 돌린다.

### 2-1. 초기 가설의 정확도를 높이는 방법

감으로 가설을 세우지 않는다. **사전 데이터로 시작점을 올린다.**

**도구/대행 가설:**

| 데이터 소스 | 수집 방법 | 가설에 주는 것 |
|-----------|----------|-------------|
| Reddit/커뮤니티 글 | AI 톤 분석 (불만/불편 감지) | **소재**: 어떤 문제가 진짜 아픈지 |
| upvote + 댓글 수 | API 자동 수집 | **수요 크기**: 공감하는 사람 수 |
| Google Trends 검색량 | API 자동 수집 | **시장 크기**: 이 문제를 검색하는 사람 수 |
| 기존 도구 리뷰/별점 | 크롤링 | **경쟁 약점**: 어디를 찌를지 (UX 가설 방향) |
| 다채널 등장 빈도 | 수집기 교차 분석 | **보편성**: 여러 곳에서 나오면 진짜 |

**콘텐츠 가설:**

| 데이터 소스 | 수집 방법 | 가설에 주는 것 |
|-----------|----------|-------------|
| 니치별 인기 쇼츠 (유튜브/틱톡) | API 수집 | **형태**: 어떤 포맷이 바이럴 되는지 |
| 인기 영상 댓글 반응 | 크롤링 | **톤**: "so true" 패턴 = 공감 소재 확인 |
| 조회수 대비 구독자 비율 | 계산 | **바이럴 지표**: 비구독자도 보는 영상 = 알고리즘 추천됨 |
| 첫 3초 패턴 | AI 분석 | **후킹**: 뭘 해야 멈추는지 |
| 영상 길이 분포 | 통계 | **최적 길이**: 몇 초짜리가 잘 되는지 |
| Pain point 커뮤니티 | AI 톤 분석 | **소재**: 어떤 주제가 공감 받는지 |

**가설 공식:**

```
도구 가설 = pain point(수요 증거) + 경쟁 약점(방향) + UX 변형(AI 생성)
콘텐츠 가설 = pain point(소재, 공감 보장) + 인기 레퍼런스(형태, 바이럴 보장)
```

### 2-2. 신뢰도 점수

가설마다 초기 신뢰도를 산출. 높은 것부터 실행:

```
신뢰도 = (수요 증거 x 0.30) + (경쟁 부재 x 0.25) + (해결 가능성 x 0.25) + (수익화 잠재력 x 0.20)

0.7 이상: 바로 제작
0.4-0.7: 추가 조사 후 판단
0.4 미만: 스킵
```

### 2-3. 검증 후 판단

| 영역 | 실험 방법 | 판단 데이터 |
|------|----------|-----------|
| 대행 타겟 | 3가지 타겟 동시 DM | 응답률 비교 |
| 대행 가격 | 3가지 가격 동시 제안 | 전환율 비교 |
| 콘텐츠 형태 | 같은 소재 x 6형태 업로드 | 조회수/체류시간 |
| 도구 UX | 같은 PP x 3-5가지 UX 배포 | 사용률/재방문율 |
| DM 메시지 | 3종 A/B/C | 응답률 |

## 3. 전략: 4트랙

### 트랙 A: 한국 대행 (현금)

비개발자 사업자의 비즈니스 문제를 Claude Code + Make API로 해결하고 돈을 받는다. 무료 1건 후 M2부터 유료 전환.

### 트랙 B: 글로벌 대행 (고단가 현금)

Upwork과 Twitter에서 AI 자동화 프로젝트 수주. 글로벌 1건($1,500-3,000)은 한국 3-4건과 같다. M3부터 시작.

### 트랙 C: 글로벌 콘텐츠 (자산)

직장생활 공감 썰을 영어 쇼츠로 생산. 세일즈 아닌 순수 엔터테인먼트. 프로필에 서비스 링크만. 얼굴 노출 없음.

- **소재**: pain point 수집에서 가져옴 (공감 보장)
- **형태**: 인기 레퍼런스 분석에서 가져옴 (바이럴 보장)
- 6가지 형태를 모두 올려서 시장이 정한다. 미리 확정하지 않음.

### 트랙 D: 마이크로 도구 팩토리 (패시브)

Pain point를 대량 수집하고, 마이크로 웹 도구를 자동 생성. AI가 UX 가설을 세우고, 시장이 검증. Sam은 주 1시간 리뷰만.

### 4트랙 시너지

```
대행 → 팩토리: 고객이 말하는 pain = 검증 완료 데이터
대행 → 콘텐츠: 대행 과정 자체가 콘텐츠 소재
팩토리 → 콘텐츠: 도구 출시마다 쇼츠 1편
콘텐츠 → 팩토리: 채널이 도구의 초기 트래픽 부스터
팩토리 → 대행: 히트 도구가 리드 생성 역할
전부 → Agent: 솔루션 DB에 누적 (장기 씨앗)
```

대행 1건 = 세 겹의 가치:
1. **돈** (단기 현금)
2. **콘텐츠 소재** (중기 자산)
3. **솔루션 DB +1** (장기)

## 6. 대행 모델: Claude Code + Make API

### 프로세스

```
고객 상담 (Sam, 30분)
→ Claude Code가 Make.com 시나리오 JSON 설계
→ Make API로 자동 생성 + 테스트
→ Sam이 결과 확인 후 전달

건당 1-2시간. 자동화 전 대비 3-7배 효율.
```

### OpenClaw Agent 구성

| Agent | 역할 | 출력 |
|-------|------|------|
| 상담 Agent | 고객 문의 분석 + 요구사항 정리 | 구조화된 요구사항 |
| 설계 Agent | Make.com 시나리오 JSON 설계 | 시나리오 JSON |
| 구현 Agent | Make API로 시나리오 생성 + 테스트 | 작동하는 워크플로우 |
| 모니터 Agent | 배포 후 동작 모니터링 | 상태 리포트 |

### 서비스 메뉴 (Reddit pain point 기반)

| 서비스 | 해결하는 Pain Point | 타겟 |
|--------|---------------------|------|
| AI 워크플로우 구축 | 도구 과부하 (73%가 90일 내 포기) | 솔로프리너/소상공인 |
| 리드 파이프라인 자동화 | 수입 불안정 (Feast-or-Famine) | 프리랜서 |
| 인보이스/결제 자동화 | 수동 인보이스 에러 | 소상공인 |
| 테크스택 다이어트 | 도구 너무 많아서 혼란 | 솔로프리너 |

### 가격 (가설 - 데이터로 검증)

```
초기 3가지 가격 동시 제안 → 전환율 비교 → 최적 가격 결정
한국: 30만 / 50만 / 80만
글로벌: $300 / $500 / $1,500
```

## 7. 마이크로 도구 팩토리

### 수집 (자동)

| 채널 | 비용 | 일일 목표 |
|------|:----:|:---------:|

**AI 톤 분석**: 키워드 매칭이 아닌, 글 전체를 읽고 불만/불편/갈증 톤 감지.

### 필터

```
"현존 AI 서비스(GPT/Claude/Gemini/Perplexity)로 즉시 해결 가능한가?"
→ Yes: 제외
→ No: 비즈니스 기회
```

**AI로 안 되는 이유 카테고리:**

| 카테고리 | 비즈니스 기회 |
|---------|-------------|
| 시스템 간 연결/통합 | 자동화 대행 or 통합 도구 |
| 실시간/지속 모니터링 | 모니터링 도구 |
| 실제 실행/대행 필요 | 구축 대행 |
| 고유 데이터 필요 | 맞춤 대시보드 |
| 인간 연결 필요 | 커뮤니티 |
| 반복/자동 실행 필요 | 자동화 도구 |

### 초기 가설 정확도 높이기

```
수요 증거 수집 (upvote, 검색량, 다채널 등장)
+ 경쟁 약점 분석 (기존 도구 리뷰에서 불만 추출)
→ 신뢰도 점수 산출 (0.7+ 만 제작)
→ UX 가설 방향도 경쟁 약점에서 도출
```

### UX 다중 가설 (자동)

```
Pain point 1개 → AI가 UX 가설 3-5개 생성 (경쟁 약점 기반 방향)
→ 전부 배포 → 시장이 승자 선택
```

```
HTML + Tailwind + Vanilla JS → Cloudflare Pages (무료, 상업적 허용)

품질 게이트: Lighthouse 90+ 미달 시 배포 차단
```

## 8. 콘텐츠 트랙

### 가설 구조: 소재(PP) x 형태(레퍼런스)

```
소재 = pain point 수집에서 (공감 보장)
형태 = 인기 레퍼런스 분석에서 (바이럴 보장)

콘텐츠 가설 = 소재 x 형태
```

### 인기 레퍼런스 수집 (자동)

```
니치별 인기 쇼츠 100개 수집 → AI가 분석:
  1. 소재 패턴: 어떤 주제가 반복? (회의, 상사, 퇴근 등)
  2. 형태 패턴: 어떤 포맷이 조회수 높나?
  3. 후킹 패턴: 첫 3초에 뭘 하나?
  4. 길이 패턴: 몇 초짜리가 잘 되나?
  5. 톤 패턴: 분노? 자조? 위트?

→ 패턴 일치도 높은 소재 x 형태 조합 = 초기 가설
```

### 콘텐츠 형태 자동 생성

| 형태 | 자동화 방법 | 비용/편 |
|------|-----------|:------:|
| 일러스트 + voice over | DALL-E + ElevenLabs + ffmpeg | $0.10 |
| 텍스트 슬라이드 + BGM | Claude(텍스트) + ffmpeg | $0.02 |
| AI 캐릭터 애니메이션 | HeyGen/D-ID API | $0.50-1.00 |
| 스톡 영상 + voice over | Pexels API + ElevenLabs | $0.10 |
| 밈/짤 + 텍스트 | DALL-E + ffmpeg | $0.05 |
| 화면 녹화 스타일 | 스크린샷 생성 + ffmpeg | $0.03 |

초기: 소재 1개 x 형태 6개 = 6편 업로드 → 승자 형태 확정 후 해당 형태로만 주 3편.

### 수집 파이프라인 공유

```
[AI 톤 분석 수집] -- 도구/콘텐츠 공유
      |
  +---+---+
  |       |
[도구]  [콘텐츠]
  |       |
PP +     PP(소재) +
경쟁약점  레퍼런스(형태)
  |       |
UX가설   소재x형태 가설
  |       |
배포     업로드
  |       |
시장검증  시장검증
```

### 콘텐츠 카테고리

| 카테고리 | 예시 | 목적 |
|---------|------|------|
| 직장 공감 썰 | "When your boss says 'quick question' at 5:59 PM" | 구독자 |
| 도구 출시 | "I built a free tool that solves X" | 도구 트래픽 |
| Build in Public | "Week 12: $500/month from micro tools" | 브랜딩 |
