# 9차 고민 — 전체 전략의 약점 공격

> 이전 차수에서 발전/수정한 점: Round 8에서 90일 실행 계획을 주 단위로 설계했다. 하지만 계획이 아무리 정교해도, 바탕이 되는 가정이 틀리면 무의미하다. 이번에는 지금까지의 모든 생각에서 가장 취약한 가정을 찾아 파괴 테스트한다.

## 핵심 질문
"이것이 틀리면 전체가 무너지는" 가정은 무엇인가? 그리고 그것이 틀릴 확률은 얼마나 되는가?

## 생각 전개

### 취약 가정 1: "솔로프리너가 ₩50~150만을 내고 세일즈 파이프라인 구축을 맡길 것이다"

**이것이 전체 전략의 토대인 이유:**
- Round 7에서 선택한 시나리오 B의 핵심 수익원 = 고단가 세팅비
- 90일 계획의 성공/실패가 이 가정에 달림
- 이 수입이 없으면 생활비 커버 불가 → 사업 지속 불가

**파괴 테스트: 왜 안 낼 수 있는가?**

```
반론 1: "내가 직접 하면 되는 거 아냐?"
→ ChatGPT, Claude가 이미 이런 것을 도와줌
→ YouTube에 "AI 세일즈 파이프라인 만들기" 검색하면 무료 영상 수십 개
→ 솔로프리너는 기본적으로 DIY 성향

반론 2: "₩50만이면 알바 2주 치인데..."
→ 한국 솔로프리너/프리랜서 중 월 수입 300만원 이하가 상당수
→ 수입의 16~50%를 "아직 검증 안 된 서비스"에 투자? → 높은 허들

반론 3: "이 사람 누군데?"
→ Sam의 인지도 0, 레퍼런스 0
→ "AI 파이프라인 구축"을 검색하면 크몽에 더 저렴한 옵션 존재
→ 브랜드가 없는 상태에서 고단가는 정당화하기 어려움

반론 4: "세일즈 파이프라인이 뭔데?"
→ 솔로프리너 대부분은 "세일즈 파이프라인"이라는 개념 자체를 모를 수 있음
→ 니즈가 잠재적(latent)이지 명시적(explicit)이지 않을 수 있음
```

**이 가정이 틀릴 확률: 40~50% (높음)**

**만약 틀리면:**
- Month 1~3에 매출 0~₩100만 → 생활비 비상금 소진
- 90일 후 피봇 필요
- 가격을 ₩20~30만으로 낮추거나, 타겟을 바꿔야 함

**사전 대비책:**
1. **무료 진단을 세일즈 퍼널로 활용** — "₩50만"을 바로 제안하지 말고, 무료 진단 → 가치 확인 → 유료 제안
2. **성과 보수 모델 병행** — "세팅은 ₩20만, 매출 증가분의 15%" → 리스크 공유
3. **첫 3건은 무료/저가** — 포트폴리오가 쌓이면 가격 올리기 쉬움
4. **크몽/숨고 등록** — 플랫폼의 신뢰를 빌려 초기 고객 확보 (단가 낮지만 사례는 쌓임)
5. **비상금 6개월치** 확보 후 시작 (BUSINESS_PLAN에서는 3개월이었는데, 6개월이 안전)

### 취약 가정 2: "Build in Public 콘텐츠가 실제로 고객을 데려올 것이다"

**이것이 중요한 이유:**
- Round 3에서 "한계비용 0 배급"의 핵심 수단 = 콘텐츠
- Round 7에서 시나리오 B의 브랜드 구축 전략 = Build in Public
- 콘텐츠가 안 되면 = 영원히 DM 아웃리치에 의존 = 스케일 불가

**파괴 테스트: 왜 안 될 수 있는가?**

```
반론 1: Build in Public은 영어권 문화
→ 한국에서 #빌드인퍼블릭 태그: 거의 없음
→ 한국 솔로프리너가 이런 콘텐츠를 소비하는 습관이 없을 수 있음
→ 한국에서는 "완성된 성공 스토리"를 선호 ("과정 공유"는 덜 매력적)

반론 2: 콘텐츠 경쟁이 극심
→ 매일 수만 개의 콘텐츠가 올라옴
→ 팔로워 0에서 시작하면 알고리즘에 노출 자체가 안 됨
→ "3개월 동안 매일 글 써도 팔로워 100명 미만" = 매우 현실적인 시나리오

반론 3: 볼 사람이 없으면 의미 없음
→ "콘텐츠 80개 누적"이라고 했는데, 팔로워 50명이면 80개나 100개나 같음
→ 콘텐츠의 질보다 배포 채널이 더 중요할 수 있음

반론 4: 공개하면 따라 한다
→ 파이프라인 구축 과정을 투명하게 공개 → 경쟁자가 그대로 복사
→ "정보 때문에 돈 내는 시대는 끝남"이라고 했는데, 과정 공개도 결국 "정보"
```

**이 가정이 틀릴 확률: 50~60% (높음) — 특히 한국 시장에서**

**만약 틀리면:**
- 3개월 동안 콘텐츠를 열심히 만들었지만 인바운드 0건
- 모든 고객 확보가 수동 아웃리치(DM)에 의존
- "노동=돈" 루프에서 벗어나지 못함

**사전 대비책:**
1. **한국에서는 Build in Public 대신 "사례 공유"로 프레이밍** — "이런 결과를 만들어냈습니다" (성공 스토리 중심)
2. **커뮤니티 침투 전략** — 네이버 카페, 디스콰이엇, 오픈카톡에 정보성 글로 존재감 확보
3. **영어 콘텐츠는 X(트위터) + LinkedIn에 집중** — Build in Public은 영어권에서만 운영
4. **숏폼 영상** — 텍스트보다 도달 범위 넓음, Before/After 비주얼은 업종 불문 매력적
5. **기대치 조정** — 콘텐츠 효과는 6~12개월 후부터. 초반 3개월은 아웃리치가 주력, 콘텐츠는 "씨앗 뿌리기"

### 취약 가정 3: "대행에서 구독/SaaS로 자연스럽게 전환될 것이다"

**이것이 중요한 이유:**
- Round 7의 3억 달성 경로에서 M12 이후의 수입 대부분이 구독(MRR)에서 발생
- 대행만으로는 3억 달성 불가능 — 구독 전환이 필수
- Round 4에서 "대행은 시작점, 그 너머가 핵심"이라고 했음

**파괴 테스트: 왜 전환이 안 될 수 있는가?**

```
반론 1: "세팅했는데 쓰지 않는다"
→ 파이프라인을 만들어줘도 실제로 운영하지 않는 고객 비율이 높을 수 있음
→ "쓰지 않는 것"에 월 ₩30만을 내지 않음
→ 세팅 완료 = 관계 종료 패턴

반론 2: "유지보수 비용 정당화 어려움"
→ "파이프라인이 잘 돌아가면 유지보수가 왜 필요하지?"
→ 고객이 "스스로 관리할 수 있다"고 느끼면 구독 취소
→ 자동화의 역설: 잘 만들수록 관리 필요성이 줄어듦

반론 3: SaaS 개발 리소스
→ 1인이 대행 + 콘텐츠 + SaaS 개발을 동시에? → 번아웃
→ SaaS MVP까지 최소 3~6개월의 개발 기간
→ 그 동안 대행 수입이 줄어들면 현금 흐름 위기

반론 4: 이미 좋은 도구가 많다
→ Zapier, Make, Notion, Airtable — 이미 충분히 좋은 도구가 존재
→ "Sam의 SaaS"를 써야 하는 이유가 불명확할 수 있음
→ 범용 도구 vs 특화 도구 경쟁에서 1인 개발자가 이기기 어려움
```

**이 가정이 틀릴 확률: 30~40% (중간)**

**만약 틀리면:**
- M12 이후에도 수입이 대행 세팅비에 의존 → 월 ₩300~500만 천장
- 3억 달성 시점이 4~5년으로 늦춰짐
- "노동=돈" 루프에서 벗어나지 못함

**사전 대비책:**
1. **"관리형 구독" 모델** — 단순 유지보수가 아니라 "매주 리포트 + 최적화 제안" 포함
2. **전환비용 만들기** — 고객 데이터가 Sam의 시스템에 쌓이도록 설계 → 떠나기 어렵게
3. **SaaS 대신 "관리형 서비스"** — 직접 SaaS를 만들지 말고, 기존 도구 조합 + Sam의 관리 = "서비스형 소프트웨어"
4. **디지털 상품으로 보완** — 구독 전환이 안 되더라도, 디지털 상품 판매로 반복 수익 확보
5. **커뮤니티가 구독을 대체** — 도구에 돈을 안 내도 커뮤니티에는 낼 수 있음

### 종합: 3가지 가정의 동시 실패 시나리오

```
최악의 경우:
✗ 고단가 세일즈 실패 (₩50만 → ₩0~20만)
✗ 콘텐츠 효과 없음 (인바운드 0)
✗ 구독 전환 안 됨 (세팅하고 끝)

→ 결과: 크몽/숨고에서 ₩10~20만짜리 건을 따내는 프리랜서 → 월 ₩100~200만
→ 3억은 10년+ 걸림
```

**하지만 3가지가 동시에 실패할 확률은 낮다.**
- 가정 1 실패(40%) × 가정 2 실패(50%) × 가정 3 실패(35%) = 7%
- 적어도 하나는 작동할 가능성이 93%

### "이것만은 반드시 맞아야 하는" 최소 조건

3가지 가정 중 **가정 1(세일즈)이 가장 중요하다.** 왜냐하면:

```
세일즈가 되면:
→ 대행 수입으로 생존 가능
→ 대행하면서 콘텐츠/데이터/관계 축적 (가정 2, 3의 기반)
→ 시간이 오래 걸려도 결국 도달

세일즈가 안 되면:
→ 현금 흐름 0 → 사업 지속 불가
→ 콘텐츠도 커뮤니티도 무의미 (매출이 없는 과정 공유는 "실패 일기")
→ 가정 2, 3을 테스트할 기회조차 없음
```

**따라서 첫 90일의 최우선 과제는 "유료 고객 3건"이다.** 나머지는 전부 부차적이다.

### 파괴 테스트 결론

| 가정 | 실패 확률 | 실패 시 영향 | 대비 수준 |
|------|----------|-------------|----------|
| 고단가 세일즈 | 40~50% | **치명적** (사업 지속 불가) | 중간 (Plan B 있음) |
| 콘텐츠 효과 | 50~60% | **심각** (스케일 불가) | 중간 (아웃리치로 보완) |
| 구독 전환 | 30~40% | **중간** (천장 있지만 생존 가능) | 약함 (대안 불명확) |

**가장 시급한 대비: 가정 1의 실패에 대한 Plan B를 구체화해야 함.**

## 이전 차수 오류 수정

Round 8에서 90일 계획의 "성공 조건"으로 "유료 고객 3건 이상, 월 수입 ₩200만 이상"을 제시했는데, 파괴 테스트 결과 이 조건이 달성되지 않을 확률이 상당하다 (40~50%).

따라서 90일 계획에 **"실패 시 빠른 피봇"을 위한 주간 체크포인트**를 추가해야 한다:
- Week 4: DM 응답률 10% 미만이면 → 메시지 전면 수정
- Week 6: 유료 계약 0건이면 → 가격을 ₩30만으로 낮추고 성과보수 모델 테스트
- Week 8: 유료 2건 미만이면 → 타겟을 소상공인으로 피봇 고려
- Week 10: 유지보수 전환 0건이면 → 관리형 구독 모델로 리디자인
