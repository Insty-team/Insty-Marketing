## Sam의 시간 현실

- 정규직: 월~금, 9시~18시 (SLAM 엔지니어)
- 가용 시간: 평일 저녁 2시간 (20:00~22:00) + 주말 각 4시간 (9:00~13:00)

- 연인/사회적 시간: 주말 오후~저녁 확보 필수

## 4. 번아웃 방지

### 현실 체크: 정규직 + 사이드 + 연인/가족 시간

| 항목 | 시간/주 |
|------|---------|
| 정규직 | 45시간 (9~18시 + 통근) |
| 사이드 프로젝트 | 18시간 |
| 수면 | 49시간 (7시간/일) |
| 식사/생활 | 14시간 |
| 운동 | 3시간 (주 3회, 1시간) |
| 연인/사회 시간 | 10시간 (주말 오후~저녁) |
| **여유 시간** | **29시간** (하루 평균 4시간) |

하루 평균 4시간의 여유 → **빡빡하지만 가능한 수준**.

### 번아웃 방지 규칙 5가지

**1. "주말 오후는 무조건 비운다" 규칙**
- 토/일 13:00 이후 = 사이드 프로젝트 금지
- 연인과의 시간, 운동, 취미, 회복에 사용
- 이 규칙을 깨는 순간 → 번아웃 경고 신호

**2. "주 1일 완전 오프" 규칙**
- 매달 1회: 토요일 또는 일요일 하루 전체 오프 (주 14시간으로 운영)
- "쉬어야 더 잘 만든다" → 자기 허락

**3. "3주 ON, 1주 LIGHT" 리듬**
- 3주: 정상 루틴 (주 18시간)
- 1주: 라이트 모드 (주 10시간) — 콘텐츠 2개만, 커뮤니티 최소 운영
- 사전 제작한 콘텐츠 버퍼로 라이트 주 커버

**4. "숫자가 안 나와도 6개월은 간다" 마인드셋**
- 첫 3개월은 숫자 보지 않기 (구독자, 조회수)
- 6개월 시점에서만 판단
- 중간에 숫자 집착 → 번아웃 + 잘못된 방향 선택

**5. 콘텐츠 버퍼 유지**
- 항상 2주 분량의 쇼츠 스크립트를 미리 작성해둠
- 예상치 못한 야근, 컨디션 난조 시 버퍼에서 충당
- 버퍼 소진 → 라이트 주 발동

## 5. 자동화 가능한 것 — Sam의 AI 에이전트 스킬 적용

### 현재 Sam이 가진 자동화 역량
- 유튜브 쇼츠 자동화 파이프라인 (경험 있음, 최대 1,500 views)
- Claude Code + Gemini API 활용
- Python 스크립트 기반 자동화
- Notion API 연동

### 자동화 파이프라인 설계

```
[콘텐츠 자동화 파이프라인]

1. 트렌드 수집 에이전트
   → YouTube/Reddit/Twitter에서 "사이드 프로젝트" 관련 인기 콘텐츠 자동 수집
   → 매일 아침 Notion DB에 저장
   → 도구: YouTube Data API + Reddit API + Gemini (요약)

2. 스크립트 생성 에이전트
   → 트렌드 데이터 + Sam의 스타일 가이드 → 쇼츠 스크립트 초안 생성
   → Sam은 10분 편집만
   → 도구: Claude API + 커스텀 프롬프트

3. 자막/편집 자동화
   → 스크립트 → TTS 또는 Sam 음성 녹음 → 자동 자막 생성
   → CapCut API or Whisper → 자막 씌우기 자동화
   → 도구: Whisper + FFmpeg + Python

4. 업로드 자동화
   → 쇼츠 영상 + 제목 + 설명 + 태그 → YouTube API로 예약 업로드
   → 도구: YouTube Data API v3

5. 뉴스레터 자동화
   → 이번 주 쇼츠 요약 + 커뮤니티 하이라이트 → 뉴스레터 초안
   → Sam은 5분 편집 + 개인 코멘트 추가
   → 도구: Gemini API + Beehiiv API

6. 커뮤니티 봇
   → 매일 질문 자동 포스팅
   → 신규 멤버 온보딩 자동 DM
   → 주간 활동 요약 자동 생성
   → 도구: Discord.py + GPT API
```

### 자동화로 절약되는 시간

| 작업 | 수동 시간 | 자동화 후 시간 | 절약 |
|------|-----------|-------------|------|
| 트렌드 리서치 | 2시간/주 | 15분/주 (리뷰만) | 1.75시간 |

### 피봇 시나리오

**시나리오 A: 구독자는 느린데 커뮤니티 반응 좋음**
→ 유튜브 쇼츠 비중 줄이고, 블로그 SEO + 뉴스레터 중심으로 전환
→ 롱폼 콘텐츠 비중 높이기

**시나리오 B: 구독자는 빠른데 유료 전환 안 됨**
→ 무료 가치가 너무 높은 것 → 유료 전용 콘텐츠 명확히 차별화
→ 가격 낮추기 (월 1.5만원부터 시작)

**시나리오 C: 한국 시장 반응 미약**
→ 영어 콘텐츠 비중 높이기 (글로벌 시장 테스트)
→ "Build in public" 영어 쇼츠/트윗 중심으로

**시나리오 D: 정규직 업무 과부하 (야근 증가)**
→ 주 18시간 → 주 10시간 라이트 모드
→ 콘텐츠 주 2개로 줄이고, 자동화에 집중
→ 커뮤니티 모더레이터 조기 도입
