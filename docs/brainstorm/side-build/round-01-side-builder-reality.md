# Round 1: 사이드 빌더의 현실 — 정규직 하면서 진짜 가능한가?

> 작성일: 2026-03-19
> 핵심 질문: "평일 저녁 + 주말로 사이드 프로젝트를 성공시킬 수 있는가?"

---

## 1. 글로벌 사이드 빌더 성공 사례

### Pieter Levels (@levelsio)
- **현재 수익**: 월 $250K+ (약 3.3억원), 연 $3M+
- **주요 제품**: NomadList, RemoteOK, PhotoAI, InteriorAI
- **타임라인**:
  - 2014년: NomadList 론칭 (디지털 노마드 DB)
  - 2019년: RemoteOK 성장 (원격 채용 보드)
  - 2023년 2월: PhotoAI 론칭 → 첫 달 $5.4K
  - 2023년 7월: PhotoAI $61.8K/월

  - 2025년 11월: $132-138K/월

- **핵심 인사이트**: 10년간 쌓은 600K+ 트위터 팔로워가 핵심 자산. PhotoAI는 "10년 청중 + AI 타이밍"의 결합
- **Sam에게 적용**: 팔로워 없이 시작하면 Pieter 경로는 불가능. 다만 "여러 제품을 만들고 되는 것에 집중"하는 전략은 유효

## 2. 한국 사이드 빌더 사례

- **제품**: 개인 금융 관리 웹앱 (구독 모델)

  - 2019년: 사이드 프로젝트로 시작 → Hacker News 1위

  - 2024년: 월 $34K

### 한국 1인 개발자들의 현실
- 사이드 프로젝트 성공률: **10% 미만**
- 일반적 목표: 월 $500~$1,000 (현실적 첫 목표)
- AI 도구(Cursor, Claude Code) 활용으로 개발 속도 3-5배 향상
- 한국 시장만 타겟하면 TAM이 작음 → **글로벌 제품 필수**

## 3. 핵심 패턴 분석

### 성공한 사이드 빌더들의 공통점

| 패턴 | 빈도 | Sam 적용 가능성 |
|------|------|----------------|
| AI 타이밍 활용 | 5/5 | ✅ AI 에이전트 스킬 보유 |
| 빠른 MVP (1-4주) | 5/5 | ✅ Claude Code로 가속 |
| 글로벌 영어 시장 | 5/5 | ⚠️ 영어 콘텐츠 필요 |
| 10년+ 청중 자산 | 3/5 | ❌ 청중 없음 |
| 여러 제품 실험 | 4/5 | ✅ 가능 |
| SEO/콘텐츠 마케팅 | 3/5 | ✅ 콘텐츠 파이프라인 스킬 |
| B2B 전환 | 3/5 | ⚠️ 시간 걸림 |

### 시간 배분 (사이드 빌더 기준)
- **Tony Dinh**: 하루 4시간 (풀타임 전환 후)
- **Danny Postma**: 주 2일 프리랜싱 + 나머지 빌딩
- **현실적 사이드**: 평일 저녁 2시간 + 주말 8시간 = **주 18시간**

### "주 18시간"으로 가능한 것

| 활동 | 소요 시간 | 주 18시간 기준 |
|------|---------|-------------|
| MVP 빌드 (간단한 SaaS) | 40-80시간 | 2-4주 |
| 랜딩 페이지 + 결제 연동 | 8-16시간 | 1주 |
| 콘텐츠 마케팅 (주 2편) | 4-6시간/주 | 지속 가능 |
| 고객 지원 + 버그 수정 | 2-4시간/주 | 지속 가능 |
| 새 기능 개발 | 8-12시간/주 | 지속 가능 |

**결론**: MVP를 2-4주에 만들고 출시하는 것은 가능. 다만 마케팅과 개발을 동시에 해야 해서 **제품은 최대한 단순해야 한다**.

## 4. "평일 저녁 + 주말로 진짜 가능한가?"에 대한 답

### 가능하다. 단, 조건이 있다.

**되는 것**:
1. 2-4주 내 MVP 출시 (Claude Code 활용하면 더 빠름)
2. 월 $500-2,000 수준의 초기 수익 (3-6개월 내)
3. 제품이 맞으면 월 $5,000-10,000까지 성장 (6-12개월)

**안 되는 것**:
1. 6개월 이상의 복잡한 제품 빌드 (정규직 병행 시 동기부여 사라짐)
2. 고객 지원이 많이 필요한 제품 (시간 부족)
3. 대규모 마케팅 (돈 없음)

### Sam에게 맞는 전략

1. **Danny Postma 모델 채택**: 정규직 유지 + 사이드 빌딩
2. **Marc Lou 철학**: 빠르게 만들고, 빠르게 검증. 4주 이상 안 쓰기
3. **Tony Dinh 교훈**: 타이밍이 중요. AI 관련 제품이면 지금이 적기
4. **글로벌 영어 시장**: 한국어로만 하면 TAM 부족
5. **여러 제품 실험**: 3-4개 빠르게 만들어보고 시장 반응 보기

## 5. 핵심 숫자 요약

| 지표 | 현실적 수치 |
|------|-----------|
| MVP까지 걸리는 시간 | 2-4주 (주 18시간 기준) |
| 첫 매출까지 | 1-3개월 |
| 월 $1,000 도달 | 3-6개월 (운 좋으면) |
| 월 $5,000 도달 | 6-18개월 |
| 월 $10,000+ 도달 | 12-24개월 |
| 성공 확률 (개별 제품) | 10-20% |
| 3개 제품 중 1개 성공 확률 | 30-50% |

### 현실 체크

- **Pieter Levels**: 10년 걸림 (2014→2024). 하지만 AI 전에는 느렸고, AI 후 폭발
- **Tony Dinh**: 2년 고전 후 AI 타이밍으로 폭발 (2021→2023)
- **Danny Postma**: 프리랜싱 병행하면서 꾸준히 빌드
- **공통점**: 모두 **1-2년의 고전 기간**이 있었다

> **Sam의 현실적 기대**: 첫 6개월은 월 0-$500. 1년 후 월 $1,000-3,000. 포기하지 않으면 2년 후부터 의미 있는 수입 가능. 핵심은 "빠른 실험 + 포기하지 않기".
