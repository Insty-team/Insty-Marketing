## 2. 인디 해커들의 시간 관리법 (조사 결과)

### 아침형 vs 저녁형

| 유형 | 방법 | 장단점 |
|------|------|--------|
| 아침형 (5-7AM) | 출근 전 60-90분 집중 | 에너지 높지만 수면 부족 위험 |
| 저녁형 (7-10PM) | 퇴근 후 2-3시간 | 피로하지만 현실적 |
| 주말 집중형 | 토 8h + 일 4h | 평일 쉬지만 주말 포기 |

**Sam에게 추천**: **저녁형 + 주말 보충**. 아침형은 SLAM 엔지니어 업무 질 저하 위험.

### 성공한 사이드 빌더들의 공통 습관

1. **타이머 사용**: 2시간 블록에 Pomodoro 25분 × 4세트 + 휴식
2. **월-금은 빌드, 주말은 마케팅**: 역할 분리로 컨텍스트 스위칭 최소화
3. **매주 같은 시간에 같은 일**: 습관화가 핵심
4. **완벽주의 포기**: "shipped is better than perfect"
5. **SNS/유튜브 시간 줄이기**: 일일 시간 추적으로 숨겨진 2-3시간 확보

## 4. 번아웃 방지 전략

### Sam의 번아웃 위험 요소

| 위험 | 원인 | 대응 |
|------|------|------|
| 이중 노동 피로 | 정규직 + 사이드 매일 | 금요일은 쉬어도 OK |
| 진전 없는 정체 | MRR 0 → 동기부여 상실 | 주간 작은 wins 추적 |
| 사회적 고립 | 퇴근 후 혼자 코딩 | 주 1회 연인/친구 시간 확보 |
| 수면 부족 | 23시 이후 코딩 | 23시 하드 스톱 |

### 번아웃 방지 규칙 (반드시 지킬 것)

1. **수면 7시간 이상**: 23시-7시. 협상 불가
2. **주 1일 완전 오프**: 일요일 오후는 사이드 금지. 연인/취미/휴식
3. **금요일 유연**: 피곤하면 쉬기. "주 14시간도 OK"
4. **2주에 1번 완전 쉬기**: 격주 주말 하루는 사이드 안 함
5. **마라톤 마인드셋**: "3년 동안 할 것이다. 이번 주 안 해도 괜찮다."
6. **작은 wins 축하**: 첫 판매, 첫 가입자, MRR $100 등 기록

### 에너지 관리

```
[높은 에너지] 토요일 오전 → 가장 어려운 코딩/아키텍처 결정
[중간 에너지] 평일 저녁 → 반복적 코딩/데이터 수집
[낮은 에너지] 일요일 오후 → 리뷰/계획/콘텐츠 (창작 아님)
```

## 5. 연인/가족 시간 확보

### Sam의 현실
- 3년 내 결혼 목표 → 연인과의 시간은 "투자"
- 사이드 빌딩 때문에 관계가 나빠지면 본말전도

### 시간 배분

| 카테고리 | 주간 시간 | 비고 |
|---------|---------|------|
| 정규직 | 45h | 출퇴근 포함 |
| 수면 | 49-56h | 7-8시간 |
| 사이드 빌딩 | 14-18h | |
| **연인/사회 시간** | **8-12h** | 주중 저녁 2회 + 일요일 오후 |
| 식사/생활 | 20h | |
| 운동/건강 | 3-5h | 주 3회 30분+ |
| 여유/취미 | 5-8h | |
| **합계** | **168h** | |

### 연인과의 소통 포인트
- "나 이것 만들고 있어" — 과정 공유 (숨기지 말 것)
- 월 수입 리포트 함께 보기 — 함께 성장하는 느낌
- "3년 후 함께 살 집을 위해" — 목표 공유
- 일요일 오후는 무조건 함께 — 일관된 약속

## 6. "주 18시간"으로 뭘 할 수 있나?

### 월 72시간의 생산성 (주 18h × 4주)

| 활동 | 시간 | 결과물 |
|------|------|--------|
| MVP 코딩 | 30-40h | 1개 마이크로 SaaS or 2개 디지털 상품 |
| 콘텐츠 마케팅 | 12-16h | 트위터 20포스트 + 블로그 2편 |
| 고객 대응 | 4-8h | 이메일/DM 응답, 피드백 반영 |
| 학습/리서치 | 4-8h | 경쟁사 분석, 새 기술 학습 |
| 계획/리뷰 | 4h | 주간 리뷰 4회 |

### AI 도구로 생산성 증폭

| 활동 | AI 없이 | AI 활용 | 절감 |
|------|--------|--------|------|
| MVP 코딩 | 80h | 30-40h | **50-60%** |
| 콘텐츠 작성 | 20h | 8-12h | **40-50%** |
| 훅 데이터 수집 | 40h | 10-15h | **60-70%** |
| 고객 이메일 | 8h | 3-4h | **50%** |

> **Claude Code 활용이 핵심**: Sam이 코딩 시간을 50% 절감할 수 있으면, 사실상 주 27시간 상당의 아웃풋.

## 8. Round 7 결론

### 핵심 메시지

**"주 18시간이면 충분하다. 단, 다음 조건을 지켜야 한다."**

1. **일관성 > 강도**: 매일 2시간을 2년 하는 것이 주말 12시간을 3개월 하는 것보다 낫다
2. **AI 도구 최대 활용**: Claude Code가 코딩 시간을 50% 줄여준다
3. **23시 하드 스톱**: 수면 7시간은 협상 불가
4. **연인 시간 확보**: 사이드 빌딩이 관계를 망치면 의미 없다
5. **2주에 1일 완전 오프**: 번아웃 방지 안전장치
6. **마라톤**: 3년 계획이다. 이번 주 못 해도 괜찮다

### Sam의 실행 가능성

| 질문 | 답 |
|------|-----|
| 주 18시간 확보 가능한가? | ✅ 가능 (SNS/유튜브 시간 전환) |
| 번아웃 없이 지속 가능한가? | ✅ 규칙 지키면 가능 |
| 연인 시간 확보 가능한가? | ✅ 일요일 오후 + 주중 2회 |
| HookGen MVP 4주 빌드 가능한가? | ✅ Claude Code로 72시간이면 충분 |
