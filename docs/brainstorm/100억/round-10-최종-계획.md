## 2. 단기 행동 계획 (0~12개월)

### Month 0~1: 시작

**이번 달 해야 할 것:**

1. **고객 인터뷰 10건**
   - 대상: SLAM 엔지니어 5명 + 솔로프레너 5명
   - 질문: "가장 시간 잡아먹는 반복 작업은?" / "돈 주고 해결하고 싶은 문제는?"
   - 도구: LinkedIn DM, 디스콰이엇, Reddit r/robotics

2. **MVP 범위 확정**
   - 후보 1: SLAM DevTools (파라미터 자동 튜닝)
   - 후보 2: AI Sales Pipeline SaaS (현재 Insty를 제품화)
   - 후보 3: AI Agent Marketplace (에이전트 거래소)
   - **고객 인터뷰 결과에 따라 결정. 추측하지 말 것.**

3. **Build in Public 시작**
   - X (Twitter) 영어 계정 개설
   - "SLAM 엔지니어 → SaaS 빌더" 스토리 공유
   - 매일 1트윗 (진행 상황, 배운 것)

### Month 1~3: MVP 개발

4. **MVP 4주 내 출시**
   - Claude Code + Cursor 활용
   - "완벽한 제품"이 아닌 "핵심 기능 1개"
   - 랜딩 페이지 + 결제 연동 + 핵심 기능

5. **첫 유료 고객 확보**
   - 목표: 10명, MRR 50~100만원
   - 채널: 인터뷰한 사람들에게 직접 제안
   - 가격: 초기 할인 (정가의 50%)

### Month 3~6: PMF 탐색

6. **고객 피드백 루프**
   - 주 1회 고객 통화
   - NPS 측정 (40+ = PMF 신호)
   - 기능 요청 우선순위화

7. **PMF 판단 (Month 6 시점)**
   - MRR 100만원+ 이고 성장 중 → 계속
   - MRR 50만원 미만 또는 정체 → pivot
   - 고객이 "이거 없으면 안 된다" 말하면 = PMF

### Month 6~12: 성장 또는 Pivot

8. **PMF 확인 시: 성장 집중**
   - 마케팅 채널 확장 (SEO, Product Hunt, 커뮤니티)
   - 가격 최적화 (A/B 테스트)
   - 두 번째 제품 기획 시작
   - 목표: MRR 300만원

9. **PMF 미확인 시: Pivot**
   - 다른 아이디어로 전환 (Round 3의 5개 아이디어)
   - MVP 4주 → 검증 8주 → 총 12주 사이클
   - 12개월에 최대 3번 시도 가능

## 6. 핵심 원칙 (매일 기억할 것)

### 원칙 1: 속도 > 완벽
- MVP는 4주 내 출시
- "이거 부끄러우면 너무 늦게 출시한 거다" — Reid Hoffman

### 원칙 2: 고객이 답이다
- 고객 인터뷰 없이 제품 만들지 말 것
- "No Market Need" = 실패 원인 1위 (42%)

### 원칙 3: 사업이 최고의 투자
- SaaS에 1원 투자 → 5~10원 밸류
- ETF에 1원 투자 → 1.1원
- 사업이 성장하는 한, 사업에 재투자

### 원칙 4: 시간은 유한하다
- 정규직 월 445만원 × 340년 = 100억 (불가능)
- SaaS ARR 10억 × 멀티플 10x = 100억 (가능)
- **구조를 바꿔야 결과가 바뀐다**

### 원칙 5: 안전망 위에서 도전
- 정규직 = 런웨이
- SLAM 스킬 = 언제든 복귀 가능
- 비상 자금 = 심리적 안정
- **배수의 진은 불필요. 안전망 위의 올인.**

## 9. 마지막 말

**100억은 "운 좋은 사람"의 이야기가 아니다.**

Round 1~9에서 분석한 것처럼:
- 100억 자산가 = 한국 상위 0.1% (~2만 가구)
- 대부분의 패턴: 사업 소득 → 자산 전환 → 복리
- Solo founder SaaS로 $1M+ ARR 달성: 확률 ~5%, median 24개월
- SLAM + AI는 현재 가장 투자받기 좋은 조합

Sam에게 유리한 점:
- **희소 스킬** (SLAM 5년)
- **AI 실전 경험** (에이전트 자동화)
- **낮은 생활비** (월 200만원)
- **안전망** (정규직 + SLAM 시장 수요)
- **비전** (개인 자생 지원 = 거대한 시장)

Sam에게 불리한 점:
- **자본 없음** (부트스트랩 강제)
- **혼자** (solo founder 리스크)
- **시간 제약** (주 22시간)

**그런데 불리한 점 모두 극복 가능하다:**
- 자본 없음 → 부트스트랩이 오히려 지분 100% (장점)
- 혼자 → AI가 2~3명분 역할 (Claude Code, Cursor)
- 시간 제약 → 15개월 후 퇴사하면 풀타임

---

**결론: Sam이 100억을 만들 확률은 0%가 아니다. 10~30%다.**

그리고 만들지 못하더라도, 이 과정에서 얻는 것(SaaS 경험, 글로벌 네트워크, 투자 자산, 자유)은 정규직만 유지했을 때보다 압도적으로 크다.

**시작하지 않으면 확률은 정확히 0%다.**

**오늘 시작하면 0%가 아니게 된다.**
